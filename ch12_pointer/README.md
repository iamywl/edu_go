**포인터(pointer)**는 메모리 주소를 저장하는 변수이다. Go에서는 포인터를 통해 값을 효율적으로 전달하고 수정할 수 있다. C/C++과 달리 포인터 연산(주소에 정수를 더하거나 빼는 행위)은 허용되지 않아 메모리 안전성이 보장된다. 포인터는 Go 프로그래밍에서 성능 최적화와 데이터 공유를 위해 필수적으로 사용되는 개념이다.

## 12.1 포인터란? (&와 * 연산자)

### & 연산자 (주소 연산자)

변수 앞에 `&`를 붙이면 해당 변수의 **메모리 주소**를 얻는다.

```go
a := 42
p := &a  // p는 a의 메모리 주소를 저장
fmt.Println(p)  // 0xc0000b6010 (메모리 주소)
```

메모리 주소는 실행할 때마다 달라질 수 있다. 운영체제가 프로그램에 할당하는 메모리 위치가 매번 다르기 때문이다.

### * 연산자 (역참조 연산자)

포인터 앞에 `*`를 붙이면 해당 주소에 저장된 **값**에 접근한다.

```go
fmt.Println(*p)  // 42 — p가 가리키는 주소의 값
*p = 100         // 주소를 통해 값을 변경!
fmt.Println(a)   // 100 — 원본이 변경됨
```

`*`는 두 가지 맥락에서 사용된다는 점에 주의해야 한다.
- **타입 선언 시**: `*int`는 "int를 가리키는 포인터 타입"이다.
- **값 접근 시**: `*p`는 "포인터 p가 가리키는 값"이다.

### 포인터 타입

```go
var p *int       // int를 가리키는 포인터 (제로값: nil)
var q *string    // string을 가리키는 포인터
```

- `*int`는 "int를 가리키는 포인터 타입"이다.
- 초기화하지 않은 포인터의 값은 `nil`이다.
- `nil` 포인터를 역참조하면 **런타임 패닉**이 발생한다. 따라서 포인터를 사용하기 전에 nil 여부를 확인하는 것이 안전하다.

## 12.2 포인터는 왜 쓰나?

### 이유 1: 큰 데이터의 복사 방지

Go에서 함수에 값을 전달하면 **복사**가 일어난다. 구조체가 크면 복사 비용이 크다.

```go
type BigData struct {
    Data [1000000]int  // 약 8MB
}

// 나쁜 예: 8MB를 통째로 복사
func processValue(d BigData) { ... }

// 좋은 예: 포인터(8바이트)만 전달
func processPointer(d *BigData) { ... }
```

포인터를 사용하면 데이터 크기에 관계없이 항상 8바이트(64비트 시스템 기준)만 전달되므로 효율적이다. 일반적으로 구조체의 크기가 클수록 포인터 전달의 이점이 커진다.

### 이유 2: 함수에서 값 변경

값 복사 방식으로는 원본을 변경할 수 없다. 포인터를 사용하면 원본을 직접 수정할 수 있다.

```go
// 값 전달 — 원본 변경 불가
func addAge(s Student) {
    s.Age++  // 복사본만 변경됨
}

// 포인터 전달 — 원본 변경 가능
func addAgePtr(s *Student) {
    s.Age++  // 원본이 변경됨!
}
```

이 차이를 이해하는 것이 중요하다. 값 전달 방식에서는 함수 내부에서 아무리 값을 변경해도 호출한 쪽의 원본에는 아무런 영향이 없다.

### 포인터 사용 가이드라인

| 상황 | 포인터 사용 여부 | 이유 |
|------|-----------------|------|
| 큰 구조체 전달 | 사용 | 복사 비용 절약 |
| 함수에서 원본 수정 필요 | 사용 | 원본 접근 필요 |
| 작은 값 타입 (int, bool 등) | 미사용 | 복사 비용이 미미함 |
| nil을 유효한 값으로 사용 | 사용 | "값 없음"을 표현 가능 |

## 12.3 인스턴스 (Instance)

**인스턴스**는 메모리에 실제로 할당된 데이터 객체이다. 구조체 타입을 정의하는 것은 "설계도"를 만드는 것이고, 구조체 변수를 생성하는 것은 설계도대로 "실제 객체"를 만드는 것이다.

### new() 함수

```go
p := new(Student)       // Student의 제로값 인스턴스를 생성
p.Name = "홍길동"        // 포인터로도 . 연산자 사용 가능!
fmt.Println(p.Name)     // "홍길동"
```

- `new(T)`는 `T` 타입의 제로값을 할당하고 `*T` 포인터를 반환한다.
- Go에서는 포인터에도 `.` 연산자를 바로 사용할 수 있다 (자동 역참조). 즉, `p.Name`은 `(*p).Name`과 동일하다.

### &struct{} 방식 (더 많이 사용)

```go
p := &Student{
    Name:  "홍길동",
    Age:   20,
    Grade: "A",
}
// p의 타입은 *Student
```

- 초기값을 지정할 수 있어 `new()`보다 더 자주 사용된다.
- `&` 연산자로 구조체 리터럴의 주소를 바로 얻는다.

### 비교

| 방식 | 초기값 지정 | 반환 타입 | 사용 빈도 |
|------|------------|----------|----------|
| `new(T)` | 불가 (제로값) | `*T` | 적음 |
| `&T{...}` | 가능 | `*T` | 많음 |

### 포인터가 같은 인스턴스를 가리키는 경우

```go
p1 := &Student{Name: "홍길동"}
p2 := p1          // p2도 같은 인스턴스를 가리킨다
p2.Name = "김철수"
fmt.Println(p1.Name)  // "김철수" — p1도 변경된다!
```

하나의 인스턴스를 여러 포인터가 가리킬 수 있다. 이 경우 어느 포인터를 통해 수정하든 같은 데이터가 변경된다.

## 12.4 스택 메모리와 힙 메모리

### 스택 (Stack)

- 함수 호출 시 자동으로 할당/해제된다.
- 매우 빠르다. LIFO(Last In, First Out) 구조로 단순한 포인터 이동만으로 할당과 해제가 가능하다.
- 함수가 끝나면 자동으로 정리된다.
- 크기가 제한되어 있다 (Go의 고루틴은 작은 스택으로 시작하여 필요 시 자동으로 확장된다).

### 힙 (Heap)

- 동적으로 할당되며, **가비지 컬렉터(GC)**가 관리한다.
- 스택보다 느리다. 메모리 할당/해제에 더 많은 처리가 필요하다.
- 함수가 끝나도 참조가 있으면 유지된다.
- GC가 주기적으로 사용되지 않는 메모리를 탐지하여 해제한다.

### Escape Analysis (탈출 분석)

Go 컴파일러는 변수가 함수 밖으로 "탈출"하는지 분석하여 스택/힙 할당을 결정한다. 이 과정을 **탈출 분석(escape analysis)**이라 한다.

```go
// 스택에 할당 — 함수 내부에서만 사용
func stackExample() int {
    x := 42
    return x  // 값을 복사하여 반환
}

// 힙에 할당 — 포인터가 함수 밖으로 탈출
func heapExample() *int {
    x := 42
    return &x  // x의 주소를 반환 → x는 힙으로!
}
```

탈출 분석 확인 명령어:
```bash
go build -gcflags="-m" main.go
```

이 명령어를 실행하면 `moved to heap: x`와 같은 메시지를 통해 어떤 변수가 힙으로 이동했는지 확인할 수 있다.

### 핵심 포인트

- Go 개발자는 `malloc`/`free`를 직접 호출하지 않는다.
- 컴파일러가 자동으로 최적의 메모리 위치를 결정한다.
- GC가 사용하지 않는 힙 메모리를 자동 해제한다.
- 불필요한 힙 할당을 줄이면 성능이 향상된다.
- 포인터를 반환하는 함수는 힙 할당을 유발할 가능성이 높다는 점을 인지해야 한다.

---

## 핵심 요약

1. `&`는 변수의 주소를 얻고, `*`는 주소의 값에 접근한다.
2. 포인터를 사용하면 큰 데이터의 복사를 방지하고 함수에서 원본을 수정할 수 있다.
3. `&T{}`가 `new(T)`보다 더 자주 사용된다.
4. Go 컴파일러의 escape analysis가 스택/힙 할당을 자동으로 결정한다.
5. nil 포인터를 역참조하면 패닉이 발생하므로 주의해야 한다.

type Node struct {

- 리스트의 길이를 반환하는 함수를 작성하라.
- 리스트를 뒤집는 함수를 작성하라.

### 과제 2: 안전한 포인터 래퍼
nil 포인터 접근을 방지하는 안전한 래퍼 함수들을 작성하라.
- `SafeDeref(p *int, defaultVal int) int`: p가 nil이면 defaultVal 반환, 아니면 *p 반환
- `SafeSet(p **int, val int)`: p가 가리키는 포인터가 nil이면 새로 할당 후 값 설정
- 여러 타입에 대해 유사한 함수를 작성하고, 공통점을 파악하라.

### 과제 3: 구조체 깊은 복사
다음과 같이 포인터 필드를 가진 구조체의 **깊은 복사(deep copy)** 함수를 구현하라.
```go
type Team struct {
    Name    string
    Members []*Person
}
```
- 단순 대입(`t2 := t1`)이 얕은 복사인 이유를 설명하라.
- Members 슬라이스와 각 Person도 새로 복사하는 `DeepCopy(t *Team) *Team` 함수를 작성하라.

### 과제 4: 트리 구조 구현
이진 트리를 포인터를 사용하여 구현하라.
```go
type TreeNode struct {
    Value       int
    Left, Right *TreeNode
}
```
- 삽입, 검색, 전위/중위/후위 순회를 구현하라.
- 트리의 높이를 구하는 함수를 작성하라.

## 프로젝트 과제

### 프로젝트 1: 메모리 풀 시뮬레이터
간단한 메모리 풀을 시뮬레이션하는 프로그램을 작성하라.
- 고정 크기의 블록을 미리 할당하는 Pool 구조체를 설계하라.
- `Alloc() *Block`과 `Free(b *Block)` 함수를 구현하라.
- 할당/해제 현황을 출력하는 기능을 추가하라.
- 포인터를 통해 블록 간 연결 리스트(free list)를 관리하라.
- 스택과 힙의 동작 원리를 이해하는 것이 이 프로젝트의 목표이다.

### 프로젝트 2: 그래프 자료구조와 경로 탐색
포인터를 활용한 그래프 자료구조를 구현하고, 두 노드 간의 경로를 탐색하는 프로그램을 작성하라.
- `GraphNode` 구조체와 인접 노드 목록(`Neighbors []*GraphNode`)을 설계하라.
- 노드 추가, 간선 추가 기능을 구현하라.
- BFS(너비 우선 탐색) 또는 DFS(깊이 우선 탐색)를 구현하여 두 노드 사이의 경로를 찾아라.
- 최단 경로를 출력하는 기능을 구현하라.
