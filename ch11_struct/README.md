Go 언어에서 **구조체(struct)**는 여러 필드를 하나로 묶어 새로운 타입을 정의하는 핵심 기능이다. 객체지향 언어의 클래스와 비슷한 역할을 하지만, 상속 대신 **조합(composition)**을 사용한다. 구조체는 Go에서 데이터를 모델링하는 가장 기본적인 수단이며, 메서드를 결합하면 객체지향적 프로그래밍도 가능하다.

---

## 11.1 선언 및 기본 사용

구조체는 `type` 키워드와 `struct` 키워드를 사용하여 정의한다.

```go
type Student struct {
    Name  string
    Age   int
    Grade string
}
```

### 구조체 변수 생성

```go
var s Student           // 제로값으로 초기화
s.Name = "홍길동"       // 필드 접근은 . 연산자
s.Age = 20
s.Grade = "A"
```

- `type` 키워드로 새로운 타입을 정의한다.
- 각 필드는 **이름**과 **타입**으로 구성된다.
- 필드 접근은 `.`(dot) 연산자를 사용한다.
- 구조체를 선언만 하고 초기화하지 않으면, 모든 필드가 해당 타입의 **제로값**으로 설정된다. 예를 들어 `string`은 `""`, `int`는 `0`, `bool`은 `false`가 된다.

---

## 11.2 구조체 변수 초기화

### 방법 1: 필드명을 지정하여 초기화

```go
s := Student{
    Name:  "김철수",
    Age:   22,
    Grade: "B",
}
```

- 필드 순서를 자유롭게 지정할 수 있다.
- 지정하지 않은 필드는 **제로값**으로 초기화된다.
- **권장하는 방식**이다. 코드 가독성이 높고, 필드가 추가되어도 기존 코드가 깨지지 않는다.

### 방법 2: 순서대로 초기화

```go
s := Student{"이영희", 21, "A"}
```

- 선언된 필드 순서와 정확히 일치해야 한다.
- 모든 필드를 빠짐없이 지정해야 한다.
- 필드가 추가되면 코드가 깨질 수 있어 **비권장**이다.
- 특히 필드가 많은 구조체에서는 어떤 값이 어떤 필드에 해당하는지 파악하기 어렵다.

### 방법 3: new() 함수 사용

```go
s := new(Student)   // *Student 타입 포인터 반환
s.Name = "박영수"
s.Age = 23
```

- `new()`는 제로값으로 초기화된 구조체의 **포인터**를 반환한다.
- 포인터와 관련된 자세한 내용은 12장에서 다룬다.

## 11.3 구조체를 포함하는 구조체

### 일반적인 포함 (has-a 관계)

```go
type Address struct {
    City   string
    Street string
}

type Person struct {
    Name    string
    Age     int
    Address Address  // 필드명을 명시
}

p := Person{
    Name: "홍길동",
    Age:  30,
    Address: Address{
        City:   "서울",
        Street: "종로",
    },
}
fmt.Println(p.Address.City) // "서울"
```

이 방식은 "Person이 Address를 **가지고 있다(has-a)**"는 관계를 명확하게 표현한다. 필드명을 통해 접근하므로 코드의 의도가 분명하다.

### 내장(Embedded) 구조체

필드명 없이 타입만 적으면 **임베딩**이 된다.

```go
type Person struct {
    Name string
    Age  int
    Address  // 필드명 생략 → 임베딩!
}

p := Person{
    Name:    "홍길동",
    Age:     30,
    Address: Address{City: "서울", Street: "종로"},
}
fmt.Println(p.City)   // "서울" — 바로 접근 가능!
fmt.Println(p.Street) // "종로"
```

- 임베딩된 구조체의 필드에 **바로 접근**할 수 있다. 이를 **필드 승격(field promotion)**이라 한다.
- 이름이 겹치면 **바깥쪽 필드가 우선**한다. 예를 들어 Person에도 `City` 필드가 있으면, `p.City`는 Person의 City를 참조한다.
- 임베딩은 상속이 아니라 **조합(composition)**이다. Go는 상속을 지원하지 않으며, 임베딩을 통해 코드 재사용을 달성한다.
- 임베딩된 구조체의 메서드도 함께 승격된다. 이 특성은 인터페이스 구현 시 매우 유용하다.

## 11.4 구조체 크기 (메모리 패딩과 정렬)

컴퓨터는 CPU가 메모리를 효율적으로 읽기 위해 **메모리 정렬(alignment)**을 수행한다. 이 과정에서 필드 사이에 빈 공간(**패딩**)이 추가될 수 있다. 메모리 정렬이 필요한 이유는, CPU가 정렬되지 않은 주소에서 데이터를 읽을 때 추가적인 메모리 접근이 필요하여 성능이 저하되기 때문이다.

```go
// 비효율적 배치: 패딩이 많이 발생
type Bad struct {
    A bool    // 1바이트 + 패딩 7바이트
    B float64 // 8바이트
    C int32   // 4바이트 + 패딩 4바이트
}
// 전체 크기: 24바이트

// 효율적 배치: 패딩 최소화
type Good struct {
    B float64 // 8바이트
    C int32   // 4바이트
    A bool    // 1바이트 + 패딩 3바이트
}
// 전체 크기: 16바이트
```

**규칙**: 큰 타입부터 작은 타입 순서로 필드를 배치하면 메모리를 절약할 수 있다.

### 정렬 규칙의 핵심

| 타입 | 크기 | 정렬 단위 |
|------|------|-----------|
| `bool`, `byte` | 1바이트 | 1바이트 |
| `int16`, `uint16` | 2바이트 | 2바이트 |
| `int32`, `uint32`, `float32` | 4바이트 | 4바이트 |
| `int64`, `uint64`, `float64`, 포인터 | 8바이트 | 8바이트 |

각 필드는 자신의 정렬 단위의 배수인 오프셋에 배치되어야 한다. 구조체 전체의 크기는 가장 큰 정렬 단위의 배수가 된다.

## 11.5 프로그래밍에서 구조체의 역할

구조체는 **관련 데이터를 하나로 묶는** 역할을 한다.

### 구조체 없이

```go
studentName := "홍길동"
studentAge := 20
studentGrade := "A"
// 변수가 흩어져 있어 관리가 어렵다
```

### 구조체 사용

```go
type Student struct {
    Name  string
    Age   int
    Grade string
}

s := Student{Name: "홍길동", Age: 20, Grade: "A"}
// 하나의 변수로 관련 데이터를 모두 관리
```

### 구조체의 장점

| 장점 | 설명 |
|------|------|
| **응집성** | 관련 데이터를 하나로 묶어 관리한다 |
| **가독성** | 코드의 의도를 명확하게 표현한다 |
| **재사용** | 함수 매개변수, 반환값으로 활용한다 |
| **확장성** | 메서드를 추가하여 기능 확장이 가능하다 |
| **타입 안전성** | 컴파일 시점에 필드 타입을 검증한다 |

### 구조체와 함수

구조체는 함수의 매개변수와 반환값으로 자주 사용된다. 여러 관련 값을 하나의 구조체로 묶어 전달하면 함수 시그니처가 간결해지고, 매개변수의 의미가 명확해진다.

```go
// 매개변수가 너무 많은 함수 (비권장)
func createUser(name string, age int, email string, phone string) { ... }

// 구조체를 사용한 함수 (권장)
func createUser(u User) { ... }
```

---

## 핵심 요약

1. 구조체는 `type 이름 struct { ... }`로 정의한다.
2. 필드명 지정 초기화(`Name: "값"`)를 권장한다.
3. 구조체 임베딩으로 필드를 승격(promote)시킬 수 있다.
4. 메모리 패딩을 고려하여 필드 순서를 배치하면 메모리를 절약할 수 있다.
5. 구조체는 관련 데이터를 묶어 코드의 가독성과 유지보수성을 높인다.

## 연습문제

### 문제 1: 도서 관리
`Book` 구조체를 정의하라. 필드: `Title`(string), `Author`(string), `Pages`(int), `Price`(float64). 책 정보를 출력하는 프로그램을 작성하라.

### 문제 2: 내장 구조체
`Engine` 구조체(HorsePower int, Type string)를 `Car` 구조체에 임베딩하라. Car에서 Engine의 필드에 직접 접근하는 코드를 작성하라.

### 문제 3: 메모리 최적화
다음 구조체의 크기를 `unsafe.Sizeof()`로 확인하고, 필드 순서를 변경하여 크기를 줄여 보라.
```go
type Data struct {
    Flag    bool
    Value   float64
    Counter int32
}
```

### 문제 4: 구조체 비교
두 개의 `Student` 구조체 변수를 만들고, `==` 연산자로 비교하라. 같은 값과 다른 값을 가진 경우를 각각 테스트하라. Go에서 구조체 비교가 가능한 조건은 무엇인지 설명하라.

### 문제 5: 빈 구조체
`struct{}`(빈 구조체)의 크기를 `unsafe.Sizeof()`로 확인하라. 빈 구조체가 Go에서 어떤 용도로 사용되는지 조사하고 예시를 작성하라. (힌트: `map[string]struct{}`, 채널 신호 등)

### 문제 6: 구조체 슬라이스
`Student` 구조체의 슬라이스를 만들고, 3명의 학생 정보를 저장하라. 전체 학생의 평균 나이를 계산하는 함수 `averageAge(students []Student) float64`를 작성하라.

### 문제 7: 중첩 구조체 초기화
다음 구조체를 정의하고, 모든 필드를 한 번에 초기화하는 코드를 작성하라.
```go
type Company struct {
    Name    string
    Address Address
    CEO     Person
}
```

### 문제 8: 필드 태그
구조체에 JSON 태그를 추가하고, `encoding/json` 패키지를 사용하여 구조체를 JSON 문자열로 변환하라. 태그에 `omitempty` 옵션을 사용하면 어떤 효과가 있는지 확인하라.

### 문제 9: 구조체 복사와 독립성
구조체 변수를 다른 변수에 대입한 후, 한쪽의 필드를 변경했을 때 다른 쪽에 영향이 있는지 확인하라. 이 결과를 통해 Go에서 구조체 대입이 어떻게 동작하는지 설명하라.

### 문제 10: 익명 구조체
별도의 타입 정의 없이 익명 구조체를 사용하여 변수를 선언하고 초기화하라. 익명 구조체가 유용한 상황은 언제인지 설명하라.

### 과제 3: 은행 계좌 시뮬레이터

### 과제 4: 2D 도형 계산기

- 두 점 사이의 거리를 계산하는 함수 작성
- 사각형의 넓이와 둘레를 계산하는 함수 작성
