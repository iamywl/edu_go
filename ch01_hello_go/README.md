## 1.1 Go의 역사

Go(또는 Golang)는 **2009년** Google에서 공개한 프로그래밍 언어이다. "Golang"이라는 이름은 공식 명칭은 아니지만, "Go"라는 단어가 너무 일반적이어서 검색 편의를 위해 널리 쓰이는 별칭이다.

### 만든 사람들

| 이름 | 약력 |
|------|------|
| **Robert Griesemer** | V8 JavaScript 엔진, Java HotSpot 컴파일러 개발에 참여한 컴파일러 전문가이다 |
| **Rob Pike** | Unix, UTF-8, Plan 9 개발에 참여한 전설적인 프로그래머이다 |
| **Ken Thompson** | Unix, B 언어, UTF-8 개발자이며, 튜링상 수상자이다 |

이 세 사람은 모두 시스템 프로그래밍 분야의 대가로, 수십 년간의 경험을 바탕으로 Go를 설계하였다. 특히 Ken Thompson은 C 언어의 전신인 B 언어를 만든 인물이기도 하다.

### Go 탄생 배경

2007년, Google의 세 엔지니어는 기존 언어들의 문제점을 해결하기 위해 새로운 언어를 설계하기 시작하였다:

- **C/C++**: 컴파일이 너무 느리고, 의존성 관리가 복잡하다. 대규모 프로젝트에서 빌드에 수십 분이 걸리는 경우도 있었다.
- **Java**: 너무 장황하고, 빌드 시스템이 복잡하다. 간단한 작업에도 많은 보일러플레이트 코드가 필요하다.
- **Python**: 실행 속도가 느리고, 타입 안전성이 부족하다. 대규모 프로젝트에서 런타임 오류를 사전에 잡기 어렵다.

Go는 이 문제들을 해결하면서도 **간결하고 효율적인** 언어를 목표로 만들어졌다. Google 내부의 대규모 소프트웨어 개발 환경에서 느낀 불편함이 Go 탄생의 직접적인 동기가 되었다.

### 주요 연혁

| 연도 | 사건 |
|------|------|
| 2007 | Go 언어 설계 시작 |
| 2009 | Go 오픈소스로 공개 |
| 2012 | Go 1.0 출시 (안정 버전). 이때부터 하위 호환성(backward compatibility)을 보장한다 |
| 2015 | Go 1.5 (셀프 호스팅 - Go로 Go 컴파일러를 작성하였다. C 의존성을 완전히 제거) |
| 2022 | Go 1.18 (제네릭 도입. Go 커뮤니티에서 가장 많이 요청한 기능이었다) |
| 2024 | Go 1.22 이후 지속적인 성능 개선과 표준 라이브러리 확장이 이루어지고 있다 |

### Go가 널리 사용되는 분야

Go는 특히 다음과 같은 분야에서 강점을 보인다:

- **클라우드 인프라**: Docker, Kubernetes, Terraform 등이 Go로 작성되었다
- **웹 서버 / API 서버**: 높은 동시성 처리 능력 덕분에 웹 백엔드 개발에 적합하다
- **CLI 도구**: 단일 바이너리 배포가 가능하여 명령줄 도구 개발에 많이 사용된다
- **네트워크 프로그래밍**: 표준 라이브러리에 강력한 네트워크 패키지가 내장되어 있다

## 1.2 Go 언어의 특징

### 정적 타입 (Static Typing)

변수의 타입이 컴파일 시점에 결정된다. 타입 오류를 실행 전에 잡을 수 있어 안전하다. 정적 타입 언어는 코드 에디터에서 자동 완성, 리팩토링 등의 도구 지원도 훨씬 강력하게 받을 수 있다는 장점이 있다.

```go
var name string = "Go"  // name은 반드시 문자열이다
var age int = 15         // age는 반드시 정수이다
// age = "열다섯"        // 컴파일 에러! int에 string을 넣을 수 없다
```

### 가비지 컬렉션 (Garbage Collection)

메모리 관리를 자동으로 해준다. C/C++처럼 수동으로 메모리를 해제할 필요가 없다. Go의 가비지 컬렉터는 저지연(low-latency)을 목표로 설계되어 있어, 대부분의 경우 프로그램 실행에 눈에 띄는 영향을 주지 않는다.

```go
func createData() {
    data := make([]int, 1000000) // 메모리 할당
    // 함수가 끝나면 가비지 컬렉터가 자동으로 메모리를 회수한다
    _ = data
}
```

### 동시성 (Concurrency)

Go의 가장 큰 강점 중 하나이다. **고루틴(goroutine)**과 **채널(channel)**을 통해 동시성 프로그래밍을 쉽게 할 수 있다. 고루틴은 운영체제 스레드보다 훨씬 가벼워서, 수천~수만 개를 동시에 실행하는 것이 가능하다.

```go
go doSomething()  // 고루틴으로 동시 실행 (나중에 자세히 배운다)
```

### 빠른 컴파일 속도

Go는 컴파일 속도가 매우 빠르다. 대규모 프로젝트도 몇 초 안에 컴파일된다. 이는 Go가 의존성 분석을 효율적으로 수행하고, 사용하지 않는 import를 허용하지 않는 등 설계 단계부터 빠른 컴파일을 고려하였기 때문이다.

### 단순한 문법

Go는 의도적으로 기능을 최소화하여 언어를 단순하게 유지한다:

- 클래스가 없다 (구조체와 메서드를 사용한다)
- 상속이 없다 (인터페이스와 컴포지션을 사용한다)
- 예외(exception)가 없다 (에러 값을 반환한다)
- 제네릭 (Go 1.18부터 지원한다)
- 세미콜론 자동 삽입 (코드 끝에 `;`가 불필요하다)

이러한 단순함 덕분에 Go 코드는 누가 작성하든 비슷한 형태를 유지하며, 팀 단위 개발에서 코드 리뷰와 유지보수가 수월하다.

## 1.3 코드가 실행되기까지

Go는 **컴파일 언어**이다. 소스 코드가 실행되기까지의 과정은 다음과 같다:

```
[소스 코드]  →  [컴파일러]  →  [실행 파일]  →  [실행]
 main.go        go build      main (바이너리)   ./main
```

### 단계별 설명

**1단계: 소스 코드 작성**
`.go` 확장자로 Go 소스 코드를 작성한다.

**2단계: 컴파일**
`go build` 명령으로 소스 코드를 기계어로 변환한다. Go 컴파일러는 소스 코드의 문법 검사, 타입 검사를 수행하고, 운영체제에 맞는 실행 파일을 생성한다.

**3단계: 실행 파일 생성**
컴파일 결과로 **단일 실행 파일(single binary)**이 만들어진다. 별도의 런타임이나 가상 머신이 필요 없다. 이것은 Go의 큰 장점 중 하나로, 실행 파일 하나만 서버에 배포하면 바로 실행할 수 있다는 것을 의미한다.

**4단계: 실행**
생성된 실행 파일을 직접 실행한다.

### 컴파일 언어 vs 인터프리터 언어

| 항목 | 컴파일 언어 (Go, C) | 인터프리터 언어 (Python, JavaScript) |
|------|---------------------|--------------------------------------|
| 실행 방식 | 소스 코드를 기계어로 변환 후 실행한다 | 소스 코드를 한 줄씩 해석하며 실행한다 |
| 실행 속도 | 빠르다 | 상대적으로 느리다 |
| 배포 | 실행 파일만 배포하면 된다 | 소스 코드 + 인터프리터가 필요하다 |
| 오류 발견 | 컴파일 시점에 많은 오류를 잡는다 | 실행 시점에 오류를 발견한다 |

### go run vs go build

```bash
# go run: 컴파일 + 실행을 한 번에 (개발 중 테스트에 편리)
go run main.go

# go build: 컴파일만 수행 (실행 파일 생성)
go build -o myapp main.go
./myapp
```

> **참고**: `go run`은 내부적으로 임시 디렉토리에 컴파일 후 실행하는 것이므로, 배포할 때는 반드시 `go build`를 사용한다.

## 1.4 Hello Go World 코드 뜯어보기

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go World!")
}
```

### 한 줄씩 분석

#### `package main`

- 모든 Go 파일은 반드시 **패키지 선언**으로 시작해야 한다.
- `main` 패키지는 특별한 패키지로, **실행 가능한 프로그램**을 만든다.
- `main`이 아닌 패키지는 라이브러리로 사용된다.
- 하나의 디렉토리 안에 있는 모든 Go 파일은 같은 패키지에 속해야 한다.

#### `import "fmt"`

- 외부 패키지를 가져오는 구문이다.
- `fmt`는 **format**의 약자로, 텍스트 입출력을 담당하는 표준 라이브러리 패키지이다.
- 사용하지 않는 패키지를 import하면 **컴파일 에러**가 발생한다 (Go의 엄격함!).
- 여러 패키지를 import할 때는 괄호로 묶어서 작성한다:

```go
import (
    "fmt"
    "math"
    "strings"
)
```

#### `func main()`

- `func`는 **함수(function)**를 정의하는 키워드이다.
- `main()` 함수는 프로그램의 **시작점(entry point)**이다. 프로그램이 실행되면 가장 먼저 호출되는 함수이다.
- `main` 패키지에는 반드시 `main()` 함수가 있어야 한다.
- `main()` 함수는 매개변수를 받지 않으며, 반환값도 없다.

#### `fmt.Println("Hello, Go World!")`

- `fmt` 패키지의 `Println` 함수를 호출한다.
- `Println`은 **Print Line**의 약자로, 문자열을 출력하고 줄바꿈한다.
- 패키지의 함수를 사용할 때는 `패키지명.함수명()` 형식으로 호출한다.
- 함수 이름이 **대문자**로 시작하면 외부에서 접근 가능(exported)하다. 소문자로 시작하면 패키지 내부에서만 사용할 수 있다(unexported).

#### 중괄호 `{}`의 위치

Go에서는 여는 중괄호 `{`가 반드시 같은 줄에 있어야 한다:

```go
// 올바른 형태
func main() {
}

// 컴파일 에러!
func main()
{
}
```

이는 Go의 세미콜론 자동 삽입 규칙 때문이다. Go 컴파일러는 특정 토큰(예: `)`, 식별자, `return` 등) 뒤에 줄바꿈이 오면 자동으로 세미콜론을 삽입한다. 따라서 `func main()` 뒤에 줄바꿈이 오면 `func main();`으로 해석되어 문법 오류가 발생하는 것이다.

## 구현 과제

1. **자기소개 프로그램**: `fmt.Println`과 `fmt.Printf`를 사용하여 이름, 나이, 거주지, 좋아하는 프로그래밍 언어를 출력하는 프로그램을 작성하라. `fmt.Printf`에서 `%s`, `%d` 서식 지정자를 활용하라.

2. **ASCII 아트 출력기**: `fmt.Println`을 사용하여 간단한 ASCII 아트(예: 집, 나무, 로켓 등)를 출력하는 프로그램을 작성하라. 최소 10줄 이상의 출력으로 구성하라.

3. **구구단 출력 프로그램**: 특정 단(예: 7단)의 구구단을 출력하는 프로그램을 작성하라. `fmt.Printf`를 사용하여 정렬된 형태로 출력하라. (아직 반복문을 배우지 않았으므로 `fmt.Println`을 여러 번 사용해도 된다.)

4. **다국어 인사 프로그램**: 한국어, 영어, 일본어, 중국어 등 최소 5개 언어로 "안녕하세요"를 출력하는 프로그램을 작성하라. Go가 UTF-8을 기본으로 지원하므로 다국어 문자열을 자유롭게 사용할 수 있다.

5. **명함 출력 프로그램**: 아래와 같은 형태의 명함을 텍스트로 출력하는 프로그램을 작성하라:
   ```
   ┌──────────────────────────────┐
   │  홍길동 (Hong Gildong)       │
   │  Software Engineer           │
   │  Email: hong@example.com     │
   │  Phone: 010-1234-5678        │
   └──────────────────────────────┘
   ```

## 프로젝트 과제

1. **Go vs 다른 언어 비교 보고서 프로그램**: Go, Python, Java, C로 각각 "Hello World"를 작성하고, 각 언어의 코드를 주석으로 포함하는 Go 프로그램을 작성하라. 프로그램은 각 언어의 특징(컴파일/인터프리터, 타입 시스템, 메모리 관리 방식)을 `fmt.Println`으로 비교 출력해야 한다.

2. **Go 퀴즈 프로그램**: 이 장에서 배운 내용을 바탕으로, Go에 대한 퀴즈를 출력하고 정답을 그 아래에 출력하는 프로그램을 작성하라. 최소 5문제를 포함하며, `fmt.Println`으로 문제와 정답을 구분하여 출력하라. (아직 입력을 배우지 않았으므로 출력만 하면 된다.)
