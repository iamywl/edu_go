## 1. 학습 개요

본 학습 계획표는 Go 언어를 체계적으로 학습하기 위한 8주(56일) 과정이다.

| 항목 | 내용 |
|------|------|
| **총 학습 기간** | 약 8주 (56일) |
| **일일 학습 시간** | 2시간 |
| **총 학습 시간** | 약 112시간 |
| **대상** | 프로그래밍 기초 지식이 있는 학습자 |
| **목표** | Go 언어 기초부터 gRPC 서버 개발까지 |

### 난이도별 시간 배분

전체 112시간은 다음과 같이 배분된다.

| 단계 | 기간 | 시간 | 비율 | 내용 |
|------|------|------|------|------|
| **1단계: 기초 문법** | 1-3주차 (21일) | 42시간 | 37.5% | 환경 구축, 변수, 함수, 제어문, 구조체, 포인터 |
| **2단계: 고급 기법** | 4-6주차 (21일) | 42시간 | 37.5% | 슬라이스, 인터페이스, 동시성, 제네릭 |

1단계와 2단계에 충분한 시간을 할당하여 기초를 탄탄히 다진 뒤, 3단계에서 실전 서버 개발에 진입하는 구조이다.

## 2. 학습 원칙

### 학습 사이클

모든 챕터는 다음 순서를 따른다.

```
README 읽기 → 예시 코드 직접 타이핑 → 실행 → 연습문제 → 구현과제 → 프로젝트 과제
```

### 핵심 원칙

1. **코드는 반드시 직접 타이핑한다.** 복사-붙여넣기는 학습 효과를 크게 떨어뜨린다. 예시 코드를 눈으로 읽고, 에디터에 한 줄씩 직접 입력하며, 각 줄의 의미를 이해한다.

2. **모르는 부분은 GO_CODING_STANDARDS.md를 참조한다.** 코딩 표준 문서에 Go의 관례와 모범 사례가 정리되어 있다. 코드를 작성하면서 수시로 참고한다.

3. **매주 복습일을 반드시 지킨다.** 각 주차의 마지막 날(Day 7, 14, 21...)은 복습일이다. 새로운 내용을 학습하지 않고, 해당 주차에 배운 내용을 정리하고 구현과제를 푼다.

4. **에러를 두려워하지 않는다.** 컴파일 에러와 런타임 에러는 최고의 학습 도구이다. 에러 메시지를 주의 깊게 읽고, 원인을 파악하는 습관을 기른다.

5. **Docker 환경을 적극 활용한다.** 본 학습 자료는 Docker 기반으로 구성되어 있다. 로컬 환경 오염 없이 일관된 학습 환경을 유지할 수 있다.

6. **하루 2시간을 지킨다.** 하루에 4시간, 5시간 몰아서 하는 것보다 매일 2시간씩 꾸준히 하는 것이 효과적이다. 과부하가 걸리면 학습 의욕이 떨어진다.

## 6. 학습 팁

### Docker 환경에서의 효율적인 학습법

1. **컨테이너를 항상 실행 상태로 유지한다.** `docker-compose up -d`로 백그라운드 실행하면 필요할 때 바로 `docker exec`로 접속할 수 있다.

2. **볼륨 마운트를 활용한다.** 로컬 에디터(VSCode 등)에서 코드를 편집하고, Docker 컨테이너에서 실행하는 워크플로가 효율적이다.

3. **Makefile 명령어를 숙지한다.** 자주 사용하는 명령어를 외워두면 학습 속도가 빨라진다.

4. **컨테이너를 지우고 다시 만드는 것을 두려워하지 않는다.** Docker의 장점은 깨끗한 환경을 언제든 재생성할 수 있다는 것이다.

### 막힐 때 대처법

1. **에러 메시지를 정확히 읽는다.** Go의 에러 메시지는 대체로 명확하다. 파일명, 줄 번호, 에러 내용을 꼼꼼히 확인한다.

2. **해당 챕터의 README를 다시 읽는다.** 대부분의 답은 README에 있다.

3. **GO_CODING_STANDARDS.md를 참조한다.** 관례적인 패턴을 모르면 코드가 동작하더라도 이상하게 느껴질 수 있다.

4. **공식 문서를 활용한다.** https://pkg.go.dev 에서 표준 라이브러리의 사용법과 예제를 확인한다.

5. **30분 이상 같은 문제에 막혀 있지 않는다.** 30분 동안 해결하지 못하면 일단 다음으로 넘어가고, 나중에 다시 돌아온다.

### 추가 학습 리소스

| 리소스 | URL | 용도 |
|--------|-----|------|
| **Go 공식 문서** | https://go.dev/doc/ | 언어 스펙, 공식 튜토리얼 |
| **Go Playground** | https://go.dev/play/ | 브라우저에서 즉시 코드 실행 |
| **Go by Example** | https://gobyexample.com/ | 예제 중심 학습 |

| **Go Blog** | https://go.dev/blog/ | 언어 설계 결정, 새 기능 소개 |
| **Go Wiki** | https://go.dev/wiki/ | 커뮤니티 지식 모음 |

### 학습 후 추천 다음 단계

1. **개인 프로젝트를 시작한다.** CLI 도구, 웹 서버, 자동화 스크립트 등 관심 분야의 프로젝트를 직접 기획하고 구현한다.

2. **오픈소스 프로젝트에 기여한다.** GitHub에서 "good first issue" 레이블이 붙은 Go 프로젝트를 찾아 기여해 본다. Docker, Kubernetes, Terraform 등이 Go로 작성되어 있다.

3. **Go 심화 주제를 학습한다.**
   - 리플렉션(reflect 패키지)
   - unsafe 패키지
   - CGO (C 라이브러리 연동)
   - 어셈블리 최적화

4. **웹 프레임워크를 심화 학습한다.** Gin, Echo, Fiber 등의 프레임워크를 사용하여 실전 규모의 웹 애플리케이션을 개발한다.

5. **데이터베이스 연동을 학습한다.** `database/sql`, GORM, sqlx 등을 이용한 데이터베이스 연동을 학습하고, CRUD 웹 애플리케이션을 구현한다.

6. **마이크로서비스 아키텍처를 학습한다.** Go는 마이크로서비스 개발에 매우 적합한 언어이다. Docker + Kubernetes 환경에서의 Go 서비스 배포를 학습한다.

7. **CNCF 프로젝트를 분석한다.** Kubernetes, Prometheus, Envoy 등 CNCF 프로젝트의 소스 코드를 읽으며 대규모 Go 프로젝트의 구조를 학습한다.
