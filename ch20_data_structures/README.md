Go 언어는 표준 라이브러리에서 다양한 자료구조를 제공한다. 이 장에서는 **리스트**, **링**, **맵**을 학습한다. 자료구조를 올바르게 선택하는 것은 프로그램의 성능과 가독성에 큰 영향을 미치므로, 각 자료구조의 특성과 적합한 사용 시점을 정확히 이해하는 것이 중요하다.

## 20.1 리스트 (container/list, 이중 연결 리스트)

### 리스트란?

`container/list` 패키지는 **이중 연결 리스트(Doubly Linked List)**를 제공한다. 각 노드가 이전 노드와 다음 노드를 가리키는 구조이다. 배열 기반 자료구조와 달리 메모리상에서 연속적이지 않으며, 각 노드가 포인터로 연결되어 있다.

### 구조 시각화

```
nil <- [A] <-> [B] <-> [C] <-> [D] -> nil
        ^                       ^
       Front                   Back
```

각 노드는 `Value` 필드에 `any` 타입의 값을 저장하므로, 어떤 타입의 데이터든 담을 수 있다. 단, 값을 꺼낼 때 타입 단언이 필요하다.

### 기본 사용법

```go
import "container/list"

l := list.New()

// 요소 추가
l.PushBack("뒤에 추가")    // 뒤에 추가
l.PushFront("앞에 추가")   // 앞에 추가

// 특정 위치에 삽입
e := l.PushBack("B")
l.InsertBefore("A", e)    // e 앞에 삽입
l.InsertAfter("C", e)     // e 뒤에 삽입

// 요소 제거
l.Remove(e)

// 순회 (앞에서 뒤로)
for e := l.Front(); e != nil; e = e.Next() {
    fmt.Println(e.Value)
}

// 역순회 (뒤에서 앞으로)
for e := l.Back(); e != nil; e = e.Prev() {
    fmt.Println(e.Value)
}
```

`PushBack()`과 `PushFront()` 등의 메서드는 삽입된 요소의 `*Element`를 반환한다. 이 포인터를 보관해두면 나중에 해당 요소를 O(1) 시간에 삭제하거나 그 위치에 새 요소를 삽입할 수 있다.

### 슬라이스 vs 리스트

| 구분 | 슬라이스 | 리스트 |
|------|----------|--------|
| 접근 | O(1) 인덱스 접근 | O(n) 순차 접근 |
| 삽입/삭제 (중간) | O(n) | O(1) (위치를 알고 있을 때) |
| 삽입/삭제 (양쪽 끝) | append O(1)* | O(1) |
| 메모리 | 연속, 캐시 친화적 | 분산 (포인터 오버헤드) |
| 타입 안전성 | 제네릭 지원 | any 타입 (타입 단언 필요) |
| 사용 시점 | 대부분의 경우 | 빈번한 중간 삽입/삭제 |

> 대부분의 경우 슬라이스가 더 좋은 성능을 보인다. 리스트는 **중간 삽입/삭제가 매우 빈번**하고, 삽입/삭제 위치의 `*Element`를 이미 알고 있을 때 유리하다. 현대 CPU의 캐시 구조를 고려하면 연속 메모리를 사용하는 슬라이스가 일반적으로 더 빠르다.

## 20.2 링 (container/ring)

### 링이란?

`container/ring` 패키지는 **원형 연결 리스트(Circular Linked List)**를 제공한다. 마지막 요소가 첫 번째 요소를 가리켜 순환 구조를 형성한다. 리스트와 달리 시작과 끝의 구분이 없으며, 어떤 노드에서 시작하든 계속 순회하면 다시 원래 위치로 돌아온다.

### 구조 시각화

```
    [A] -> [B]
     ^      |
     |      v
    [D] <- [C]
```

### 기본 사용법

```go
import "container/ring"

// 크기 5인 링 생성
r := ring.New(5)

// 값 설정
for i := 0; i < r.Len(); i++ {
    r.Value = i
    r = r.Next()
}

// 순회 (Do 메서드 사용)
r.Do(func(val any) {
    fmt.Println(val)
})

// 수동 순회
p := r
for i := 0; i < r.Len(); i++ {
    fmt.Println(p.Value)
    p = p.Next()
}
```

`ring.New(n)`은 n개의 노드를 가진 링을 생성하며, 각 노드의 `Value`는 `nil`로 초기화된다. 링의 크기는 생성 시 결정되며, `Link()`와 `Unlink()` 메서드를 통해 노드를 추가하거나 제거할 수 있다.

### 링의 주요 메서드

```go
r := ring.New(5)

r.Next()           // 다음 노드
r.Prev()           // 이전 노드
r.Move(3)          // 3칸 앞으로 이동
r.Move(-2)         // 2칸 뒤로 이동
r.Len()            // 링의 길이
r.Link(other)      // 다른 링과 연결
r.Unlink(3)        // 현재 노드 다음 3개 노드 제거
```

### 활용 사례

1. **라운드 로빈 스케줄링**: 여러 작업을 순환하며 처리한다. 서버 로드 밸런싱에서 요청을 여러 서버에 순환 분배할 때 사용한다.
2. **최근 N개 기록 유지**: 로그, 히스토리 등을 고정 크기로 유지한다. 새 항목이 추가되면 가장 오래된 항목이 자동으로 덮어써진다.
3. **순환 버퍼**: 고정 크기 버퍼에서 오래된 데이터를 자동으로 덮어쓴다. 실시간 데이터 스트리밍에서 최근 데이터만 유지할 때 유용하다.
4. **턴제 게임**: 여러 플레이어가 순서대로 턴을 진행하는 게임에서 사용한다.

## 20.3 맵 (map[K]V, CRUD 연산)

### 맵이란?

맵은 **키-값(key-value) 쌍**을 저장하는 자료구조이다. 다른 언어의 딕셔너리(Dictionary), 해시맵(HashMap)과 같다. 키를 통해 값에 빠르게 접근할 수 있으며, 평균적으로 O(1)의 시간 복잡도를 가진다.

### 맵 선언과 초기화

```go
// 1. 리터럴로 생성
scores := map[string]int{
    "김철수": 85,
    "이영희": 92,
    "박민수": 78,
}

// 2. make()로 생성
ages := make(map[string]int)

// 3. make()로 초기 용량 지정 (성능 최적화)
largeMap := make(map[string]int, 1000)

// 4. nil 맵 (읽기만 가능, 쓰기 시 패닉)
var m map[string]int  // nil
```

`make()`로 초기 용량을 지정하면, 맵이 내부적으로 충분한 버킷을 미리 확보하여 재해싱(rehashing) 횟수를 줄일 수 있다. 대략적인 데이터 크기를 알고 있다면 용량을 지정하는 것이 성능에 유리하다.

### CRUD 연산

```go
m := make(map[string]int)

// Create (생성)
m["사과"] = 1000
m["바나나"] = 2000

// Read (읽기)
price := m["사과"]       // 1000
price, ok := m["포도"]   // 0, false (없는 키)

// Update (수정)
m["사과"] = 1500

// Delete (삭제)
delete(m, "바나나")
```

존재하지 않는 키를 읽으면 해당 값 타입의 제로 값이 반환된다. 정수형이면 0, 문자열이면 "", 불리언이면 false가 반환된다. 이 때문에 "값이 0인 것"과 "키가 존재하지 않는 것"을 구분하려면 반드시 comma ok 패턴을 사용해야 한다.

### 맵 순회

```go
for key, value := range m {
    fmt.Printf("%s: %d\n", key, value)
}

// 키만 순회
for key := range m {
    fmt.Println(key)
}
```

> **주의**: 맵의 순회 순서는 **보장되지 않는다**. 매번 다른 순서로 출력될 수 있다. Go 런타임은 의도적으로 맵 순회 순서를 무작위화하여, 순서에 의존하는 코드를 방지한다. 순서가 필요하면 키를 슬라이스에 저장하고 정렬한 후 순회해야 한다.

### 존재 여부 확인 (comma ok 패턴)

```go
if val, ok := m["키"]; ok {
    fmt.Println("존재:", val)
} else {
    fmt.Println("존재하지 않음")
}
```

이 패턴은 Go에서 매우 자주 사용되는 관용적 표현이다. `ok`가 `true`이면 키가 존재하고, `false`이면 존재하지 않는다.

### 맵의 값으로 슬라이스/구조체 사용

```go
// 값이 슬라이스인 맵 (그래프의 인접 리스트 표현에 유용)
graph := map[string][]string{
    "서울": {"부산", "대전", "인천"},
    "부산": {"서울", "대구"},
}

// 값이 구조체인 맵
type Student struct {
    Name  string
    Score int
}
students := map[int]Student{
    1: {"김철수", 85},
    2: {"이영희", 92},
}
```

맵의 값이 구조체일 때 주의할 점이 있다. 맵에서 꺼낸 구조체의 필드를 직접 수정할 수 없다. `students[1].Score = 90`은 컴파일 에러가 발생한다. 구조체 전체를 꺼내서 수정한 뒤 다시 저장하거나, 값을 포인터(`map[int]*Student`)로 저장해야 한다.

## 20.4 맵의 원리 (해시 테이블)

### 해시 테이블이란?

맵은 내부적으로 **해시 테이블(Hash Table)**을 사용한다. 해시 테이블은 키를 해시 함수에 통과시켜 얻은 값으로 데이터의 저장 위치를 결정하는 자료구조이다.

```
키 "사과" -> 해시 함수 -> 해시값 42 -> 버킷[42] -> 값 1000
키 "바나나" -> 해시 함수 -> 해시값 17 -> 버킷[17] -> 값 2000
```

### 동작 원리

1. **해시 함수**: 키를 해시값(정수)으로 변환한다. 같은 키는 항상 같은 해시값을 생성해야 한다.
2. **버킷**: 해시값을 기반으로 데이터를 저장할 위치를 결정한다. 해시값을 버킷 수로 나눈 나머지가 버킷 인덱스가 된다.
3. **충돌 처리**: 같은 버킷에 여러 키가 매핑될 때 처리한다. Go는 체이닝 방식(각 버킷에 오버플로 버킷을 연결)을 사용한다.

Go의 맵은 로드 팩터(load factor)가 일정 수준을 넘으면 자동으로 버킷 수를 늘리고 모든 데이터를 재배치(rehashing)한다. 이 과정은 비용이 크므로, 초기 용량을 적절히 설정하면 성능을 향상시킬 수 있다.

### 시간 복잡도

| 연산 | 평균 | 최악 |
|------|------|------|
| 읽기 | O(1) | O(n) |
| 쓰기 | O(1) | O(n) |
| 삭제 | O(1) | O(n) |
| 검색 | O(1) | O(n) |

최악의 경우는 모든 키가 같은 버킷에 매핑될 때 발생하지만, 좋은 해시 함수를 사용하면 실질적으로 거의 발생하지 않는다.

### 맵의 키 타입 조건

맵의 키로 사용할 수 있는 타입은 제한되어 있다:
- 비교 가능한(comparable) 타입만 가능하다
- `==` 연산자로 비교할 수 있어야 한다
- **가능**: `int`, `string`, `float64`, `bool`, 구조체(필드가 모두 비교 가능한 경우), 포인터, 배열
- **불가능**: 슬라이스, 맵, 함수

`float64`를 키로 사용하는 것은 가능하지만 권장하지 않는다. 부동소수점 비교의 부정확성 때문에 예상치 못한 동작이 발생할 수 있기 때문이다.

### 맵 사용 시 주의사항

```go
// 1. nil 맵에 쓰기 -> 패닉!
var m map[string]int
// m["key"] = 1  // panic: assignment to entry in nil map

// 2. 맵은 참조 타입이다 (함수에 전달하면 원본이 수정됨)
func addItem(m map[string]int) {
    m["new"] = 100  // 원본 맵이 수정됨
}

// 3. 동시성 안전하지 않다 (여러 고루틴에서 동시 접근 시 문제)
// sync.Map 또는 뮤텍스 사용 필요

// 4. 맵의 구조체 값을 직접 수정할 수 없다
// m[key].field = value  // 컴파일 에러
// 해결: 포인터를 값으로 사용하거나, 꺼내서 수정 후 다시 저장
```

|---------|--------|------|----------|
| 리스트 | `container/list` | 이중 연결 리스트, O(1) 삽입/삭제 | 빈번한 중간 삽입/삭제 |

## 연습문제

### 문제 1: 리스트 활용
`container/list`를 사용하여 정수를 **정렬된 순서로 삽입**하는 함수를 작성하라. 삽입 후에도 리스트가 항상 정렬된 상태를 유지해야 한다.

### 문제 2: 링 활용
크기가 5인 링을 만들어 **최근 5개의 로그 메시지**를 유지하는 프로그램을 작성하라. 새 메시지가 추가되면 가장 오래된 메시지가 덮어쓰여야 한다.

### 문제 3: 맵 기본
학생 이름(string)을 키, 점수(int)를 값으로 하는 맵을 만들고 CRUD 연산을 수행하라. 전체 평균 점수도 계산하라.

### 문제 4: 단어 빈도수
문자열에서 각 단어의 출현 빈도를 계산하는 프로그램을 작성하라. `map[string]int`를 사용하여 가장 많이 등장한 단어를 찾아라.

### 문제 5: 그래프 표현
맵을 사용하여 간단한 그래프(인접 리스트)를 표현하라. `map[string][]string` 타입을 사용하여 도시 간 연결을 저장하고, 특정 도시에서 도달 가능한 모든 도시를 출력하라.

### 문제 6: 맵 정렬 출력
`map[string]int` 타입의 맵을 키의 알파벳 순서로 정렬하여 출력하는 함수를 작성하라. 값 기준으로 정렬하는 함수도 만들어라.

### 문제 7: 집합(Set) 연산
맵을 사용하여 집합(Set)을 구현하라. `map[string]bool` 또는 `map[string]struct{}` 타입을 사용하여 합집합(Union), 교집합(Intersection), 차집합(Difference) 연산을 수행하는 함수들을 작성하라.

### 문제 8: 리스트 역순
`container/list`에 저장된 데이터를 역순으로 출력하는 함수와, 리스트 자체를 역순으로 뒤집는 함수를 각각 작성하라.

### 문제 9: 이중 맵
`map[string]map[string]int` 타입을 사용하여, 학생별 과목별 점수를 저장하고 조회하는 프로그램을 작성하라. 학생별 총점, 과목별 평균 등을 계산하라. 중첩 맵을 안전하게 초기화하는 방법에 주의하라.

### 문제 10: comma ok 패턴 활용
맵에 값을 추가할 때 이미 존재하는 키인 경우 기존 값에 더하고, 새로운 키인 경우 새로 추가하는 `AddOrUpdate(m map[string]int, key string, val int)` 함수를 작성하라. 이를 활용하여 문자열에서 각 문자의 출현 횟수를 세는 프로그램을 작성하라.

## 구현 과제

### 과제 1: LRU 캐시
맵과 리스트(`container/list`)를 조합하여 **LRU(Least Recently Used) 캐시**를 구현하라. 다음 인터페이스를 만족해야 한다:
- `NewLRUCache(capacity int) *LRUCache` : 용량이 정해진 캐시 생성
- `Get(key string) (string, bool)` : 값 조회 (접근 시 최근 사용으로 갱신)
- `Put(key, value string)` : 값 저장 (용량 초과 시 가장 오래전에 사용된 항목 제거)
- `Len() int` : 현재 저장된 항목 수

맵은 O(1) 조회를 위해, 리스트는 사용 순서를 추적하기 위해 사용한다.

### 과제 2: 간단한 데이터베이스
맵을 사용하여 인메모리 키-값 데이터베이스를 구현하라. 다음 기능을 포함하라:
- SET, GET, DELETE 명령어 지원
- KEYS 패턴 매칭 (간단한 접두사 매칭)
- EXPIRE 기능 (TTL 설정, 만료된 키 자동 삭제)
- 데이터 통계 (총 키 수, 메모리 사용량 추정)

### 과제 3: 우선순위 큐
맵과 슬라이스를 조합하여 우선순위 큐를 구현하라. `Enqueue(item string, priority int)`, `Dequeue() (string, bool)`, `Peek() (string, bool)` 메서드를 포함하라. 우선순위가 높은 항목이 먼저 나와야 한다.

### 과제 4: 트라이(Trie) 자료구조
맵(`map[rune]*TrieNode`)을 사용하여 트라이(접두사 트리)를 구현하라. 다음 메서드를 포함하라:
- `Insert(word string)` : 단어 삽입
- `Search(word string) bool` : 단어 존재 여부 확인
- `StartsWith(prefix string) bool` : 접두사로 시작하는 단어가 있는지 확인
- `AutoComplete(prefix string) []string` : 접두사로 시작하는 모든 단어 반환

### 과제 5: 링 버퍼 기반 메트릭 수집기
`container/ring`을 사용하여 시계열 메트릭 수집기를 구현하라. 최근 N개의 측정값을 유지하면서 다음 기능을 제공하라:
- `Record(value float64)` : 측정값 기록
- `Average() float64` : 평균 계산
- `Min() float64`, `Max() float64` : 최소/최대값
- `MovingAverage(window int) float64` : 이동 평균
- `Percentile(p float64) float64` : 백분위수 계산

## 프로젝트 과제

### 프로젝트 1: 전화번호부 애플리케이션
맵, 슬라이스, 정렬을 종합적으로 활용하여 전화번호부 애플리케이션을 구현하라. 다음 기능을 포함해야 한다:
- 연락처 추가/수정/삭제 (이름, 전화번호, 이메일, 그룹)
- 이름으로 검색 (부분 일치 지원)
- 그룹별 연락처 조회 (`map[string][]Contact`)
- 전체 연락처 정렬 출력 (이름순, 그룹순)
- 중복 연락처 탐지 (같은 전화번호를 가진 연락처)
- 연락처 통계 (그룹별 인원수, 총 연락처 수)
- 최근 검색 기록 유지 (링 버퍼 활용)

모든 자료구조(`map`, `list`, `ring`)를 적절히 활용하여 구현하라.

### 프로젝트 2: 텍스트 분석기
맵과 슬라이스를 활용하여 텍스트 분석기를 구현하라. 긴 텍스트를 입력받아 다음 분석 결과를 제공해야 한다:
- 단어 빈도수 분석 (상위 N개 단어 출력)
- 문자 빈도수 분석
- 문장 수, 단어 수, 문자 수 통계
- N-gram 분석 (연속된 N개 단어의 출현 빈도)
- 특정 단어가 등장하는 위치(인덱스) 목록
- 역인덱스(inverted index) 구축: 각 단어가 어떤 문장에 등장하는지 매핑
- 유사 단어 추천 (간단한 편집 거리 기반)

맵의 중첩 사용(`map[string]map[string]int` 등)과 효율적인 자료구조 선택에 신경 쓰라.
