## 15.1 해법 (전체 설계)

### 게임 규칙

1. 컴퓨터가 1~100 사이의 랜덤 숫자를 선택한다.
2. 사용자가 숫자를 입력한다.
3. 입력한 숫자가 정답보다 크면 "더 작은 수입니다!" 를 출력한다.
4. 입력한 숫자가 정답보다 작으면 "더 큰 수입니다!" 를 출력한다.
5. 정답을 맞추면 시도 횟수와 함께 축하 메시지를 출력한다.
6. 최대 10번까지 시도 가능하다.

### 전체 흐름

```
프로그램 시작
  → 랜덤 숫자 생성 (1~100)
  → 반복문 시작 (최대 10회)
    → 사용자 입력 받기
    → 비교
      → 크다/작다 힌트 출력
      → 정답이면 축하 메시지 출력 후 종료
  → 10회 초과 시 정답 공개
```

## 15.2 사전지식

### rand 패키지 (난수 생성)

```go
import "math/rand"

// Go 1.20+: 자동 시드 설정
n := rand.Intn(100)  // 0~99 사이 랜덤 정수
n = rand.Intn(100) + 1  // 1~100 사이 랜덤 정수
```

> **참고**: Go 1.20 이상에서는 `rand.Seed()`를 호출할 필요가 없다. 자동으로 랜덤 시드가 설정된다. 이전 버전에서는 `rand.Seed(time.Now().UnixNano())`로 시드를 설정해야 매번 다른 난수가 생성되었다.

### 주요 rand 함수

| 함수 | 설명 | 반환값 |
|------|------|--------|
| `rand.Intn(n)` | 0 이상 n 미만의 정수 | int |
| `rand.Float64()` | 0.0 이상 1.0 미만의 실수 | float64 |
| `rand.Int()` | 음이 아닌 랜덤 정수 | int |

### fmt.Scan (사용자 입력)

```go
var input int
fmt.Print("숫자를 입력하세요: ")
fmt.Scan(&input)  // 사용자 입력을 변수에 저장
```

- `fmt.Scan(&변수)`: 공백/줄바꿈으로 구분된 값을 읽는다.
- `&`를 사용하여 변수의 **포인터**를 전달한다 (12장에서 배운 내용!).
- 입력이 올바르지 않으면 에러를 반환한다.
- 반환값으로 읽은 항목 수와 에러를 받을 수 있다.

```go
n, err := fmt.Scan(&input)
if err != nil {
    fmt.Println("입력 에러:", err)
}
```

### fmt.Scanf (포맷 지정 입력)

```go
var name string
var age int
fmt.Scanf("%s %d", &name, &age)
```

### 잘못된 입력 후 버퍼 정리

`fmt.Scan`은 입력 실패 시 입력 버퍼에 잘못된 데이터가 남을 수 있다. 이 경우 `bufio.Reader`를 사용하여 버퍼를 비워야 한다.

```go
reader := bufio.NewReader(os.Stdin)

// 잘못된 입력 시 버퍼 정리
if _, err := fmt.Scan(&guess); err != nil {
    reader.ReadString('\n')  // 남은 입력 비우기
    continue
}
```

## 15.3 랜덤한 숫자 생성하기

```go
package main

import (
    "fmt"
    "math/rand"
)

func main() {
    // 1~100 사이의 랜덤 숫자 생성
    answer := rand.Intn(100) + 1
    fmt.Printf("정답: %d (개발용, 완성 후 삭제)\n", answer)
}
```

### 범위 지정 공식

`rand.Intn(max - min + 1) + min`

| 원하는 범위 | 코드 |
|------------|------|
| 1~100 | `rand.Intn(100) + 1` |
| 1~6 (주사위) | `rand.Intn(6) + 1` |
| 10~50 | `rand.Intn(41) + 10` |

이 공식의 원리를 이해하는 것이 중요하다. `rand.Intn(n)`은 0부터 n-1까지의 값을 반환한다. 여기에 min을 더하면 min부터 min+n-1까지의 범위가 된다. n에 `max - min + 1`을 대입하면 min부터 max까지의 범위를 얻을 수 있다.

## 15.4 숫자값 입력받기

```go
var guess int
fmt.Print("숫자를 입력하세요 (1-100): ")
_, err := fmt.Scan(&guess)
if err != nil {
    fmt.Println("올바른 숫자를 입력해주세요.")
    continue  // 다시 입력받기
}
```

### 입력 검증

잘못된 입력(문자 등)을 처리하는 것이 중요하다:

```go
if guess < 1 || guess > 100 {
    fmt.Println("1~100 사이의 숫자를 입력해주세요.")
    continue
}
```

입력 검증은 프로그램의 안정성을 위해 필수적이다. 사용자는 예상치 못한 값을 입력할 수 있으며, 이를 적절히 처리해야 프로그램이 비정상적으로 종료되지 않는다.

### 이진 탐색 전략

이 게임의 최적 전략은 **이진 탐색(binary search)**이다. 매번 가능한 범위의 중간값을 추측하면 최대 7번(log2(100) ≈ 6.6) 만에 정답을 맞출 수 있다. 게임을 하면서 이진 탐색의 원리를 자연스럽게 체험할 수 있다.

## 연습문제

### 문제 1: 범위 공식 이해
`rand.Intn(max - min + 1) + min` 공식에서, 다음 범위를 생성하는 코드를 각각 작성하라.
- 1~6 (주사위)
- 20~30
- -10~10
- 100~999 (3자리 수)

### 문제 2: 입력 검증 강화
`fmt.Scan`으로 정수를 입력받되, 문자열이 입력되었을 때 프로그램이 멈추지 않고 다시 입력을 요청하도록 하는 코드를 작성하라. 입력 버퍼 정리(`bufio.Reader`)도 포함하라.

### 문제 3: 시도 횟수 분석
1~100 범위에서 이진 탐색 전략을 사용하면 최대 몇 번의 시도로 정답을 맞출 수 있는지 수학적으로 계산하라. 이를 검증하는 프로그램을 작성하라.

### 문제 4: 게임 통계
게임을 10번 자동 실행하여 각 게임의 시도 횟수를 기록하고, 평균 시도 횟수를 계산하는 프로그램을 작성하라. (사용자 입력 대신 컴퓨터가 이진 탐색으로 추측)

### 문제 5: 역할 반전
사용자가 숫자를 생각하고, 컴퓨터가 맞추는 프로그램을 작성하라. 컴퓨터는 이진 탐색 전략을 사용한다. 사용자는 "크다/작다/정답" 중 하나를 입력한다.

### 문제 6: 주사위 게임
1~6 범위의 난수 두 개를 생성하여 주사위 게임을 만들라. 사용자와 컴퓨터가 각각 주사위를 던져 큰 수가 이기는 규칙이다. 5판 3선승 방식으로 구현하라.

### 문제 7: 로또 번호 생성기
1~45 사이의 중복 없는 숫자 6개를 생성하는 로또 번호 생성기를 작성하라. 숫자는 오름차순으로 정렬하여 출력하라.

### 문제 8: 구조체 활용
게임 상태를 `GameState` 구조체(Answer, Attempts, MaxAttempts, GuessHistory []int)로 관리하도록 숫자 맞추기 게임을 리팩토링하라.

### 문제 9: 힌트 강도 조절
정답과의 차이에 따라 힌트의 강도를 다르게 출력하는 기능을 구현하라.
- 차이 50 이상: "매우 멀다!"
- 차이 20 이상: "멀다!"
- 차이 10 이상: "가깝다!"
- 차이 5 미만: "매우 가깝다!"

### 문제 10: fmt.Scanf 활용
`fmt.Scanf`를 사용하여 "이름 나이" 형태로 입력받는 프로그램을 작성하라. 게임 시작 시 사용자의 이름과 나이를 입력받아, 게임 결과 메시지에 이름을 포함하여 출력하라.

본문에서 설명한 숫자 맞추기 게임을 완성하라. 다음 요소를 포함해야 한다.

- 상수를 적절히 활용할 것

### 과제 2: 난이도 선택 기능
게임 시작 시 난이도를 선택할 수 있는 기능을 추가하라.
- 쉬움: 1~50 범위, 15번 시도 가능
- 보통: 1~100 범위, 10번 시도 가능
- 어려움: 1~200 범위, 7번 시도 가능
- 지옥: 1~1000 범위, 10번 시도 가능
- 난이도에 따라 안내 메시지를 다르게 출력하라.

### 과제 3: 재도전 및 기록 시스템
게임 종료 후 다시 도전할 것인지 묻는 기능과, 게임 기록을 관리하는 기능을 추가하라.
- 재도전 시 "y/n" 입력
- 총 플레이 횟수, 승리 횟수, 승률 출력
- 최소 시도 횟수 기록(최고 기록) 관리 및 갱신 시 알림
- 각 게임의 시도 횟수 이력 저장

### 과제 4: 두 플레이어 모드
두 명의 플레이어가 교대로 숫자를 맞추는 모드를 구현하라.
- 게임 시작 시 1인 모드 / 2인 모드를 선택한다.
- 2인 모드에서는 각 플레이어가 교대로 숫자를 입력한다.
- 먼저 정답을 맞추는 플레이어가 승리한다.
- 각 플레이어의 이름을 입력받아 턴마다 표시한다.

### 과제 5: 힌트 시스템 구현
5회 시도 후에도 정답을 맞추지 못하면 추가 힌트를 제공하는 시스템을 구현하라.
- 5회 실패: 정답이 짝수인지 홀수인지 알려준다.
- 7회 실패: 정답의 십의 자리 숫자를 알려준다.
- 9회 실패: 정답 범위를 ±5로 좁혀서 알려준다.

## 프로젝트 과제

- 메인 메뉴에서 게임을 선택할 수 있도록 구현한다.
- 각 게임의 전적을 기록하고, 전체 통계를 볼 수 있는 메뉴를 추가한다.
- 게임 로직은 별도의 함수로 분리한다.
- (선택) 패키지를 분리하여 구현한다 (14장에서 배운 내용 활용).

### 프로젝트 2: 자동 플레이어 AI
숫자 맞추기 게임을 자동으로 플레이하는 AI 프로그램을 구현하라.
- 랜덤 전략, 순차 전략, 이진 탐색 전략의 3가지 AI를 구현한다.
- 각 전략으로 1000번의 게임을 자동 실행한다.
- 전략별 평균 시도 횟수, 최소/최대 시도 횟수, 성공률을 비교 분석한다.
- 결과를 표 형태로 출력한다.
- 이 프로젝트를 통해 알고리즘의 효율성 차이를 체감할 수 있다.
