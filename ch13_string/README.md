Go의 문자열은 **UTF-8로 인코딩된 바이트 시퀀스**이다. 불변(immutable)이며, 내부적으로 포인터와 길이로 구성된 가벼운 구조이다. Go는 문자열을 언어 차원에서 기본 타입으로 지원하며, UTF-8 인코딩을 기본으로 채택하여 다국어 처리에 강점을 가진다.

## 13.1 문자열 (string 타입, UTF-8)

```go
s := "안녕하세요, Go!"
fmt.Println(s)         // 안녕하세요, Go!
fmt.Println(len(s))    // 19 (바이트 수, 글자 수가 아님!)
```

### 핵심 특징

- Go의 문자열은 **UTF-8** 인코딩을 사용한다.
- `len()`은 **바이트 수**를 반환한다 (한글 1글자 = 3바이트).
- 글자 수를 알려면 `utf8.RuneCountInString()`을 사용한다.
- 백틱(`` ` ``)으로 **Raw 문자열**을 만들 수 있다 (이스케이프 무시).

```go
raw := `줄바꿈도
그대로 유지되는
Raw 문자열`

escaped := "이스케이프: \n\t 탭"
```

### UTF-8 인코딩 이해

UTF-8은 가변 길이 인코딩 방식이다. 각 문자를 1~4바이트로 표현한다.

| 문자 범위 | 바이트 수 | 예시 |
|-----------|----------|------|
| ASCII (U+0000~U+007F) | 1바이트 | 'A', '1', '!' |
| 라틴 확장 (U+0080~U+07FF) | 2바이트 | 'ñ', 'ü' |
| 한글, 한자 등 (U+0800~U+FFFF) | 3바이트 | '가', '한', '漢' |
| 이모지 등 (U+10000~U+10FFFF) | 4바이트 | 이모지 문자들 |

Go의 창시자 중 한 명인 Rob Pike가 UTF-8 인코딩의 공동 설계자이다. 이 때문에 Go는 UTF-8을 언어 수준에서 강력하게 지원한다.

### byte와 rune 타입

```go
var b byte = 'A'    // byte = uint8, ASCII 문자 1개
var r rune = '가'   // rune = int32, 유니코드 코드포인트 1개
```

- `byte`는 `uint8`의 별칭이며, 1바이트 값을 나타낸다.
- `rune`은 `int32`의 별칭이며, 하나의 유니코드 코드포인트를 나타낸다.
- 작은따옴표(`'`)는 단일 문자(rune) 리터럴에 사용한다.

## 13.2 문자열 순회 (byte vs rune)

### byte 단위 순회

```go
s := "Go 한글"
for i := 0; i < len(s); i++ {
    fmt.Printf("%d: %x\n", i, s[i])  // 바이트 단위
}
```

- 영문은 1바이트, 한글은 3바이트로 처리된다.
- 한글이 깨져서 출력된다.
- 인덱스로 직접 접근하면 바이트 단위로 접근하게 되므로, 다국어 문자열에서는 주의가 필요하다.

### rune 단위 순회 (for range)

```go
s := "Go 한글"
for i, r := range s {
    fmt.Printf("%d: %c (U+%04X)\n", i, r, r)
}
```

출력:
```
0: G (U+0047)
1: o (U+006F)
2:   (U+0020)
3: 한 (U+D55C)
6: 글 (U+AE00)
```

- `for range`는 **rune(유니코드 코드포인트)** 단위로 순회한다.
- `rune`은 `int32`의 별칭이다.
- 인덱스는 **바이트 위치**를 나타낸다 (3 → 6으로 점프). 한글 1글자가 3바이트를 차지하기 때문이다.

### []rune 변환을 통한 인덱스 접근

```go
s := "Go 한글"
runes := []rune(s)
fmt.Println(runes[3])          // 54620 (유니코드 값)
fmt.Printf("%c\n", runes[3])  // 한
fmt.Println(len(runes))        // 5 (글자 수)
```

`[]rune`으로 변환하면 인덱스가 글자 단위가 되어, 다국어 문자열에서도 안전하게 특정 위치의 문자에 접근할 수 있다.

## 13.3 문자열 합치기

### + 연산자

```go
s := "Hello" + ", " + "World!"
```

- 간단하지만, 반복적인 합치기에서는 **비효율적**이다.
- 매번 새로운 문자열을 생성하기 때문이다. 문자열이 불변이므로 합칠 때마다 기존 내용을 새로운 메모리에 복사해야 한다.

### strings.Builder (대량 합치기에 효율적)

```go
var builder strings.Builder
for i := 0; i < 1000; i++ {
    builder.WriteString("Go")
}
result := builder.String()
```

- 내부 버퍼를 사용하여 메모리 할당을 최소화한다.
- **대량의 문자열 합치기에 권장**된다.
- `WriteString`, `WriteByte`, `WriteRune` 등의 메서드를 제공한다.

### strings.Join (슬라이스 합치기)

```go
words := []string{"Go", "is", "awesome"}
result := strings.Join(words, " ")  // "Go is awesome"
```

- 문자열 슬라이스를 구분자로 연결할 때 사용한다.
- 내부적으로 전체 길이를 미리 계산하여 한 번에 메모리를 할당하므로 효율적이다.

### fmt.Sprintf (포맷 문자열)

```go
name := "홍길동"
age := 25
s := fmt.Sprintf("%s님은 %d세입니다.", name, age)
```

다양한 타입의 값을 포맷에 맞춰 문자열로 조합할 때 유용하다.

### 성능 비교

| 방식 | 사용 상황 | 성능 |
|------|----------|------|
| `+` 연산자 | 소수의 문자열 합치기 | 보통 |
| `strings.Builder` | 루프 내 대량 합치기 | 우수 |
| `strings.Join` | 슬라이스 합치기 | 우수 |
| `fmt.Sprintf` | 포맷 지정 필요 시 | 보통 |

## 13.4 문자열 구조 (String Header)

Go의 string은 내부적으로 다음 두 필드를 가진 **구조체**이다:

```
┌─────────────────┐
│  Pointer (8B)   │ → 실제 바이트 배열을 가리킴
├─────────────────┤
│  Length  (8B)   │ → 바이트 수
└─────────────────┘
```

- string 변수의 크기는 항상 **16바이트** (64비트 시스템)이다.
- 문자열 내용이 아무리 길어도 변수 자체는 16바이트이다.
- 문자열을 함수에 전달하면 이 16바이트 헤더만 복사된다 (효율적!).

```go
s := "Hello, World!"
fmt.Println(unsafe.Sizeof(s))  // 16
```

### 문자열 대입과 메모리

```go
s1 := "Hello"
s2 := s1  // 헤더(16B)만 복사 — 같은 바이트 배열을 공유!
```

두 변수가 같은 바이트 배열을 가리키지만, 문자열이 불변이므로 한쪽을 수정할 수 없어 문제가 발생하지 않는다. 이것이 불변 문자열의 핵심적인 이점이다.

### 슬라이스([]byte)와의 차이

| 특성 | string | []byte |
|------|--------|--------|
| 구조 | 포인터 + 길이 | 포인터 + 길이 + 용량 |
| 크기 | 16바이트 | 24바이트 |
| 가변성 | 불변 | 가변 |
| 용도 | 텍스트 데이터 | 바이너리 데이터, 문자열 수정 |

## 13.5 문자열은 불변이다 (Immutability)

```go
s := "Hello"
// s[0] = 'h'  // 컴파일 에러! 문자열은 수정 불가
```

### 문자열을 수정하려면?

`[]byte`로 변환하여 수정 후 다시 `string`으로 변환한다.

```go
s := "Hello"
b := []byte(s)    // 문자열 → 바이트 슬라이스 (복사 발생)
b[0] = 'h'
s2 := string(b)   // 바이트 슬라이스 → 문자열 (복사 발생)
fmt.Println(s2)   // "hello"
```

`string`과 `[]byte` 간 변환 시 매번 데이터 복사가 발생한다는 점에 유의해야 한다. 빈번한 변환은 성능에 영향을 줄 수 있다.

### 한글 수정 시에는 []rune 사용

```go
s := "안녕하세요"
r := []rune(s)     // rune 슬라이스로 변환
r[0] = '만'
s2 := string(r)
fmt.Println(s2)    // "만녕하세요"
```

한글처럼 멀티바이트 문자를 수정할 때는 반드시 `[]rune`으로 변환해야 한다. `[]byte`로 변환하면 한글 1글자가 3바이트로 분리되어 올바른 수정이 불가능하다.

### 불변의 장점

| 장점 | 설명 |
|------|------|
| **안전성** | 여러 곳에서 같은 문자열을 공유해도 안전하다 |
| **해시 키** | map의 키로 사용 가능하다 (변경되지 않으므로) |
| **동시성** | 고루틴 간 안전하게 공유 가능하다 |
| **최적화** | 컴파일러가 문자열 리터럴을 재사용할 수 있다 |

## 13.6 유용한 strings 패키지 함수

Go 표준 라이브러리의 `strings` 패키지는 문자열 처리에 유용한 함수들을 제공한다.

```go
import "strings"

strings.Contains("Hello, World", "World")   // true
strings.HasPrefix("Hello", "He")             // true
strings.HasSuffix("Hello", "lo")             // true
strings.ToUpper("hello")                     // "HELLO"
strings.ToLower("HELLO")                     // "hello"
strings.TrimSpace("  hello  ")               // "hello"
strings.Replace("aaa", "a", "b", 2)          // "bba"
strings.Split("a,b,c", ",")                  // ["a", "b", "c"]
strings.Count("cheese", "e")                 // 3
strings.Index("Hello", "ll")                 // 2
```

이 함수들은 문자열을 다루는 거의 모든 프로그램에서 활용되므로, 어떤 함수들이 있는지 파악해 두는 것이 좋다.

## 연습문제

### 문제 1: 글자 수 세기
한글과 영문이 섞인 문자열의 바이트 수와 글자(rune) 수를 각각 출력하라.
- 입력: `"Hello 세계"`
- 출력: `바이트 수: 12, 글자 수: 8`

### 문제 2: 문자열 뒤집기
한글을 포함한 문자열을 뒤집는 `reverseString(s string) string` 함수를 작성하라.
- 입력: `"Hello 세계"`
- 출력: `"계세 olleH"`

### 문제 3: 성능 비교
`+` 연산자와 `strings.Builder`로 각각 "Go"를 10만 번 합치는 시간을 `time.Now()`로 측정하여 비교하라.

### 문제 4: 팰린드롬 판별
주어진 문자열이 팰린드롬(앞에서 읽으나 뒤에서 읽으나 같은 문자열)인지 판별하는 `isPalindrome(s string) bool` 함수를 작성하라. 한글도 올바르게 처리해야 한다.
- `"토마토"` → `true`
- `"level"` → `true`
- `"hello"` → `false`

### 문제 5: 단어 빈도 세기
문자열에서 각 단어가 몇 번 등장하는지 `map[string]int`로 집계하는 함수를 작성하라.
- 입력: `"go is great and go is fast"`
- 출력: `go:2, is:2, great:1, and:1, fast:1`

### 문제 6: 문자열 압축
연속된 같은 문자를 "문자+개수"로 압축하는 함수를 작성하라. 압축 결과가 원본보다 길면 원본을 반환하라.
- 입력: `"aaabbbccca"`
- 출력: `"a3b3c3a1"`

### 문제 7: CamelCase 변환
스네이크 케이스(snake_case) 문자열을 카멜 케이스(CamelCase)로 변환하는 함수를 작성하라.
- 입력: `"hello_world_go"`
- 출력: `"HelloWorldGo"`

### 문제 8: 바이트 단위 vs rune 단위 비교
다음 문자열에 대해 `len()`, `utf8.RuneCountInString()`, `[]rune` 변환 후 길이를 각각 출력하고 결과가 다른 이유를 설명하라.
- `"Hello"`, `"안녕하세요"`, `"Go 언어 🚀"`

### 문제 9: 문자열 치환 함수
`strings` 패키지를 사용하지 않고, 문자열에서 특정 부분 문자열을 다른 문자열로 치환하는 `myReplace(s, old, new string) string` 함수를 직접 구현하라.

### 문제 10: Raw 문자열과 이스케이프
Raw 문자열(백틱)과 일반 문자열(쌍따옴표)의 차이를 보여주는 프로그램을 작성하라. 파일 경로, JSON 문자열, 정규표현식 패턴 등 Raw 문자열이 유용한 3가지 실제 사례를 코드로 작성하라.

## 구현 과제

### 과제 1: 단어 카운터 프로그램
사용자로부터 여러 줄의 텍스트를 입력받아 다음 통계를 출력하는 프로그램을 작성하라.
- 총 문자(rune) 수
- 총 단어 수
- 총 줄 수
- 가장 긴 단어와 가장 짧은 단어
- 가장 많이 등장한 단어 Top 3

### 과제 2: 시저 암호
시저 암호(Caesar cipher)를 구현하라. 영문 알파벳을 N칸 이동시켜 암호화/복호화한다.
- `encrypt(text string, shift int) string` 함수 작성
- `decrypt(text string, shift int) string` 함수 작성
- 대소문자를 유지하고, 영문이 아닌 문자(공백, 숫자, 한글 등)는 그대로 둔다.
- 예: `encrypt("Hello, World!", 3)` → `"Khoor, Zruog!"`

### 과제 3: 마크다운 파서 (간이 버전)
간단한 마크다운 텍스트를 HTML로 변환하는 프로그램을 작성하라.
- `# 제목` → `<h1>제목</h1>`
- `## 소제목` → `<h2>소제목</h2>`
- `**굵게**` → `<b>굵게</b>`
- `*기울임*` → `<i>기울임</i>`
- 빈 줄은 `<br>` 태그로 변환

### 과제 4: 문자열 유사도 측정
두 문자열의 유사도를 측정하는 프로그램을 작성하라.
- 레벤슈타인 거리(Levenshtein distance)를 계산하는 함수를 구현하라.
- 두 문자열을 입력받아 거리와 유사도 퍼센트를 출력하라.
- 한글 문자열도 올바르게 처리해야 한다.

## 프로젝트 과제

### 프로젝트 1: 텍스트 분석기
텍스트 파일을 읽어 종합적인 분석 결과를 출력하는 프로그램을 구현하라.
- 파일 경로를 인자로 받아 내용을 읽는다.
- 글자 수, 단어 수, 문장 수, 단락 수를 계산한다.
- 단어 빈도 분석 (상위 10개 단어 출력)
- 평균 단어 길이, 평균 문장 길이를 계산한다.
- 한글과 영문의 비율을 분석한다.
- `strings` 패키지의 다양한 함수를 활용하고, `strings.Builder`로 결과 보고서를 생성하라.

### 프로젝트 2: 간이 검색 엔진
여러 문서(문자열 슬라이스)에서 키워드를 검색하는 간이 검색 엔진을 구현하라.
- 문서 등록 기능 (제목 + 본문)
- 키워드 검색 시 해당 키워드가 포함된 문서 목록을 반환한다.
- 검색 결과를 관련도 순으로 정렬한다 (키워드 등장 횟수 기준).
- 대소문자 구분 없는 검색을 지원한다.
- 검색 키워드 주변의 문맥(앞뒤 20자)을 함께 출력하라.
