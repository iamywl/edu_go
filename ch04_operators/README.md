# Chapter 04: 연산자

Go 언어에서 사용하는 다양한 연산자를 배운다. 연산자는 변수나 값에 대해 계산, 비교, 논리 판단 등의 작업을 수행하는 기호이다. Go는 다른 언어에 비해 연산자의 종류가 적고 동작이 명확하여, 코드를 읽을 때 혼동이 적다.

---

Docker 컨테이너 내부에서 다음과 같이 실행한다:

```bash

```

## 4.1 산술 연산자

기본적인 수학 연산을 수행하는 연산자이다.

| 연산자 | 설명 | 예시 | 결과 |
|--------|------|------|------|
| `+` | 덧셈 | `10 + 3` | `13` |
| `-` | 뺄셈 | `10 - 3` | `7` |
| `*` | 곱셈 | `10 * 3` | `30` |
| `/` | 나눗셈 | `10 / 3` | `3` (정수), `3.333...` (실수) |
| `%` | 나머지 | `10 % 3` | `1` |

이 산술 연산자들은 같은 타입의 피연산자에 대해서만 사용할 수 있다. 예를 들어 `int`와 `float64`를 직접 더할 수 없으며, 명시적 타입 변환이 필요하다.

### 정수 나눗셈 주의!

```go
// 정수 / 정수 = 정수 (소수점 이하 버림)
fmt.Println(10 / 3)   // 3 (3.333이 아니다!)

// 실수 나눗셈을 하려면 하나라도 실수여야 한다
fmt.Println(10.0 / 3) // 3.3333333333333335
```

정수 나눗셈에서 소수점 이하가 버려지는 것은 "내림(floor)"이 아니라 "0으로 향한 절삭(truncation toward zero)"이다. 양수의 경우 내림과 같지만, 음수의 경우 결과가 다를 수 있다:

```go
fmt.Println(-10 / 3)  // -3 (내림이면 -4이겠지만, 0 방향으로 절삭하므로 -3이다)
fmt.Println(-10 % 3)  // -1 (나머지의 부호는 피제수를 따른다)
```

### 문자열 연결

`+` 연산자는 문자열을 연결할 때도 사용한다:

```go
greeting := "Hello, " + "Go!"  // "Hello, Go!"
```

문자열 연결에서 주의할 점은, 반복적으로 `+`를 사용하여 문자열을 이어붙이면 매번 새로운 문자열이 생성되므로 성능이 좋지 않다는 것이다. 많은 문자열을 연결할 때는 `strings.Builder`를 사용하는 것이 효율적이다.

### 증감 연산자

Go에는 `++`와 `--` 연산자가 있지만, **문장(statement)**으로만 사용 가능하다:

```go
i := 0
i++    // i = i + 1 (가능)
i--    // i = i - 1 (가능)

// j := i++  // 컴파일 에러! Go에서는 식(expression)으로 사용 불가하다
// ++i       // 컴파일 에러! 전위 증감 없다
```

Go에서 `++`와 `--`를 식이 아닌 문장으로만 허용하는 이유는 C/C++에서 `a[i++] = b[++j]` 같은 코드가 만들어내는 모호함과 복잡성을 제거하기 위해서이다. 이 설계 덕분에 Go 코드는 더 명확하고 읽기 쉬워진다.

## 4.2 비교 연산자

두 값을 비교하여 `bool` 값(`true` 또는 `false`)을 반환한다.

| 연산자 | 설명 | 예시 | 결과 |
|--------|------|------|------|
| `==` | 같음 | `10 == 10` | `true` |
| `!=` | 다름 | `10 != 3` | `true` |
| `<` | 작음 | `3 < 10` | `true` |
| `>` | 큼 | `10 > 3` | `true` |
| `<=` | 작거나 같음 | `3 <= 3` | `true` |
| `>=` | 크거나 같음 | `10 >= 3` | `true` |

```go
a, b := 10, 20
fmt.Println(a == b)  // false
fmt.Println(a != b)  // true
fmt.Println(a < b)   // true
fmt.Println(a > b)   // false
```

비교 연산자도 같은 타입의 값끼리만 비교할 수 있다. `int`와 `float64`를 직접 비교하면 컴파일 에러가 발생한다.

### 문자열 비교

문자열도 비교 연산자를 사용할 수 있다 (사전순 비교, 정확히는 유니코드 코드 포인트 순서이다):

```go
fmt.Println("apple" < "banana")  // true (사전순으로 apple이 앞이다)
fmt.Println("Go" == "Go")       // true
fmt.Println("Go" == "go")       // false (대소문자를 구분한다)
```

문자열 비교에서 대문자가 소문자보다 앞에 온다. 이는 유니코드 코드 포인트에서 대문자(A=65)가 소문자(a=97)보다 작은 값을 갖기 때문이다. 대소문자를 무시하고 비교하려면 `strings.EqualFold()` 함수를 사용한다.

## 4.3 실수 오차

컴퓨터는 실수를 **부동소수점(floating-point)** 방식으로 저장한다. 이 과정에서 미세한 오차가 발생할 수 있다. 이 개념은 모든 프로그래밍 언어에 공통적으로 적용되는 것이며, Go에만 국한된 문제가 아니다.

### 문제 상황

```go
var a float64 = 0.1
var b float64 = 0.2
var c float64 = 0.3

fmt.Println(a + b)      // 0.30000000000000004 (0.3이 아니다!)
fmt.Println(a + b == c) // false!
```

### 왜 이런 일이?

0.1을 2진수로 표현하면 **무한소수**가 된다:

```
0.1 (10진수) = 0.0001100110011001100... (2진수, 무한 반복)
```

컴퓨터는 유한한 비트로 이 값을 저장해야 하므로 어딘가에서 잘라야 하고, 이때 오차가 발생한다. IEEE 754 표준에 따라 `float64`는 52비트의 가수부(mantissa)를 가지며, 이 범위 내에서 최대한 정확하게 값을 근사한다. 하지만 0.1, 0.2, 0.3 같은 간단한 10진 소수도 2진수로는 정확히 표현할 수 없는 것이다.

### 해결 방법 1: 오차 범위(epsilon) 비교

```go
import "math"

a := 0.1
b := 0.2
c := 0.3

// 두 값의 차이가 매우 작으면 같다고 판단한다
epsilon := 1e-10
if math.Abs((a+b)-c) < epsilon {
    fmt.Println("같다") // 이 결과가 출력된다
}
```

epsilon 값을 얼마로 설정할지는 상황에 따라 다르다. 일반적으로 `1e-9`에서 `1e-15` 사이의 값을 사용한다. 너무 작으면 오차를 잡지 못하고, 너무 크면 실제로 다른 값을 같다고 판단할 수 있다.

### 해결 방법 2: math.Nextafter 활용

`math.Nextafter`는 한 숫자 다음으로 표현 가능한 가장 가까운 float64 값을 반환한다:

```go
fmt.Println(math.Nextafter(0.3, 0.4))   // 0.3 다음 표현 가능한 값
fmt.Println(0.1 + 0.2)                   // 이 값과 비교해볼 수 있다
```

### 해결 방법 3: 정수로 변환하여 계산

돈이나 정밀한 계산이 필요할 때는 정수 단위로 변환한다:

```go
// 1000원 + 2000원 (원 단위의 정수로 계산한다)
price1 := 1000 // 10.00달러를 센트 단위로
price2 := 2000 // 20.00달러를 센트 단위로
total := price1 + price2
fmt.Printf("합계: $%.2f\n", float64(total)/100) // $30.00
```

금융 애플리케이션이나 결제 시스템에서는 실수형을 사용하면 안 된다. 반드시 정수(원, 센트 등 최소 단위)로 계산하거나, `math/big` 패키지의 `Rat` 타입(유리수)을 사용해야 정확한 결과를 얻을 수 있다.

## 4.4 논리 연산자

`bool` 값에 대해 논리적 연산을 수행한다.

| 연산자 | 설명 | 예시 | 결과 |
|--------|------|------|------|
| `&&` | AND (그리고) | `true && false` | `false` |
| `\|\|` | OR (또는) | `true \|\| false` | `true` |
| `!` | NOT (부정) | `!true` | `false` |

### 진리표

#### AND (`&&`): 둘 다 참이어야 참이다

| A | B | A && B |
|---|---|--------|
| true | true | **true** |
| true | false | false |
| false | true | false |
| false | false | false |

#### OR (`||`): 하나라도 참이면 참이다

| A | B | A \|\| B |
|---|---|---------|
| true | true | **true** |
| true | false | **true** |
| false | true | **true** |
| false | false | false |

### 단축 평가 (Short-circuit Evaluation)

Go의 논리 연산자는 **단축 평가**를 수행한다. 이는 결과가 이미 결정되면 나머지 표현식을 평가하지 않는다는 뜻이다:

```go
// && : 왼쪽이 false이면 오른쪽은 평가하지 않는다
false && someFunction()  // someFunction()은 호출되지 않는다

// || : 왼쪽이 true이면 오른쪽은 평가하지 않는다
true || someFunction()   // someFunction()은 호출되지 않는다
```

단축 평가는 성능 최적화뿐만 아니라, 안전한 코드를 작성하는 데도 활용된다. 예를 들어:

```go
// 슬라이스가 nil이 아닌지 먼저 확인하고, 그 다음에 길이를 검사한다
// 단축 평가 덕분에 s가 nil이면 len(s)는 평가되지 않아 panic이 발생하지 않는다
if s != nil && len(s) > 0 {
    // s를 안전하게 사용할 수 있다
}
```

### 활용 예시

```go
age := 25
hasLicense := true

// 20세 이상이고 면허증이 있어야 운전 가능하다
canDrive := age >= 20 && hasLicense
fmt.Println("운전 가능:", canDrive) // true

// 학생이거나 65세 이상이면 할인한다
isStudent := true
isSenior := false
discount := isStudent || isSenior
fmt.Println("할인 대상:", discount) // true
```

## 4.5 대입 연산자

변수에 값을 대입하는 연산자이다.

| 연산자 | 설명 | 동일 표현 |
|--------|------|----------|
| `=` | 대입 | - |
| `+=` | 더하고 대입 | `a = a + b` |
| `-=` | 빼고 대입 | `a = a - b` |
| `*=` | 곱하고 대입 | `a = a * b` |
| `/=` | 나누고 대입 | `a = a / b` |
| `%=` | 나머지 후 대입 | `a = a % b` |
| `&=` | 비트 AND 후 대입 | `a = a & b` |
| `\|=` | 비트 OR 후 대입 | `a = a \| b` |
| `^=` | 비트 XOR 후 대입 | `a = a ^ b` |
| `<<=` | 왼쪽 시프트 후 대입 | `a = a << b` |
| `>>=` | 오른쪽 시프트 후 대입 | `a = a >> b` |

```go
x := 10
x += 5   // x = 15
x -= 3   // x = 12
x *= 2   // x = 24
x /= 4   // x = 6
x %= 4   // x = 2
```

복합 대입 연산자(`+=`, `-=` 등)는 코드를 간결하게 만들어 준다. `a = a + b`보다 `a += b`가 더 읽기 쉽고, 타이핑도 적다. Go에서는 `=`와 `:=`를 혼동하지 않도록 주의해야 한다. `=`는 기존 변수에 값을 대입하는 것이고, `:=`는 새 변수를 선언하면서 동시에 값을 대입하는 것이다.

## 4.6 비트 연산자

비트 연산자는 정수의 개별 비트를 조작한다. 저수준 프로그래밍, 권한 관리, 플래그 처리 등에서 사용된다.

| 연산자 | 설명 | 예시 |
|--------|------|------|
| `&` | AND (둘 다 1이면 1) | `0b1100 & 0b1010` → `0b1000` |
| `\|` | OR (하나라도 1이면 1) | `0b1100 \| 0b1010` → `0b1110` |
| `^` | XOR (다르면 1) | `0b1100 ^ 0b1010` → `0b0110` |
| `&^` | AND NOT (비트 클리어) | `0b1100 &^ 0b1010` → `0b0100` |
| `<<` | 왼쪽 시프트 | `1 << 3` → `8` |
| `>>` | 오른쪽 시프트 | `8 >> 2` → `2` |

`&^`는 Go만의 독특한 연산자로, 다른 언어에는 없다. "비트 클리어(bit clear)" 연산이라고도 하며, 오른쪽 피연산자에서 1인 비트 위치를 왼쪽 피연산자에서 0으로 만든다.

왼쪽 시프트(`<<`)는 2를 곱하는 것과 같고, 오른쪽 시프트(`>>`)는 2로 나누는 것과 같다:

```go
fmt.Println(5 << 1)  // 10 (5 * 2)
fmt.Println(5 << 2)  // 20 (5 * 4)
fmt.Println(20 >> 1) // 10 (20 / 2)
fmt.Println(20 >> 2) // 5  (20 / 4)
```

## 4.7 연산자 우선순위

Go의 연산자 우선순위 (위가 높은 순):

| 우선순위 | 연산자 |
|----------|--------|
| 5 (최고) | `*`, `/`, `%`, `<<`, `>>`, `&`, `&^` |
| 4 | `+`, `-`, `\|`, `^` |
| 3 | `==`, `!=`, `<`, `<=`, `>`, `>=` |
| 2 | `&&` |
| 1 (최저) | `\|\|` |

### 괄호를 사용한 우선순위 제어

```go
// 괄호가 없으면 연산자 우선순위에 따라 계산된다
result1 := 2 + 3 * 4     // 2 + 12 = 14

// 괄호로 우선순위를 변경한다
result2 := (2 + 3) * 4   // 5 * 4 = 20
```

> **팁**: 복잡한 수식에서는 괄호를 적극 활용하여 의도를 명확히 하는 것이 좋다. 코드의 가독성이 높아지고, 우선순위를 외우지 않아도 되므로 버그도 줄어든다.

## 연습문제

1. 두 정수를 입력받아 사칙연산(+, -, *, /)과 나머지(%) 결과를 모두 출력하는 프로그램을 작성하라.
2. 섭씨 온도를 입력받아 화씨로 변환하는 프로그램을 작성하라 (공식: F = C * 9/5 + 32). 정수 나눗셈에 주의하라.
3. `0.1 + 0.2 == 0.3`이 `false`인 이유를 코드로 확인하고, epsilon 비교로 `true`가 되도록 수정하라.
4. 나이를 입력받아 다음 조건을 판별하는 프로그램을 작성하라:
   - 13세 미만: 어린이
   - 13세 이상 20세 미만: 청소년
   - 20세 이상 65세 미만: 성인
   - 65세 이상: 노인
5. 비트 연산자(`&`, `|`, `^`, `<<`, `>>`)를 사용하여 숫자를 조작해 보라. 각 연산의 결과를 2진수로 출력하여 동작을 확인하라.
6. 다음 표현식의 결과를 연산자 우선순위에 따라 계산하고, 실제 Go 코드로 확인하라:
   ```
   2 + 3 * 4 - 1
   (2 + 3) * (4 - 1)
   10 % 3 + 5 / 2
   true || false && false
   ```
7. 단축 평가를 활용하여, 0으로 나누기 에러를 방지하는 조건문을 작성하라. (힌트: `b != 0 && a/b > 5` 형태)
8. 정수 `-7`을 `3`으로 나눈 몫과 나머지를 출력하라. Go의 나머지 연산이 음수에 대해 어떻게 동작하는지 확인하라.
9. `&^` (AND NOT) 연산자를 사용하여 특정 비트를 끄는 예제를 작성하라. 예를 들어, `0b1111`에서 두 번째 비트를 끄는 코드를 작성하라.
10. 두 변수의 값을 임시 변수 없이 교환하는 코드를 XOR 비트 연산을 사용하여 작성하라. (힌트: `a = a ^ b; b = a ^ b; a = a ^ b`)

## 구현 과제

1. **BMI 계산기**: 키(cm)와 몸무게(kg)를 입력받아 BMI를 계산하고, 결과에 따라 저체중/정상/과체중/비만을 판별하는 프로그램을 작성하라 (BMI = 몸무게(kg) / 키(m)^2). 정수 나눗셈 문제에 주의하라.

2. **윤년 판별기**: 연도를 입력받아 윤년인지 판별하는 프로그램을 작성하라. 윤년 조건은 다음과 같다: (1) 4로 나누어 떨어지고, (2) 100으로 나누어 떨어지지 않거나, (3) 400으로 나누어 떨어지면 윤년이다. 논리 연산자를 활용하라.

3. **비트 플래그 권한 시스템**: 읽기(1), 쓰기(2), 실행(4) 권한을 비트 연산으로 관리하는 프로그램을 작성하라. 권한을 설정(`|`), 확인(`&`), 제거(`&^`)하는 기능을 모두 구현하고, 각 단계에서 권한 상태를 2진수로 출력하라.

4. **실수 정밀도 테스트기**: 다양한 실수 연산(0.1+0.2, 1.0/3.0*3.0, 등)을 수행하고, 각 결과를 소수점 20자리까지 출력하여 어디서 오차가 발생하는지 시각적으로 보여주는 프로그램을 작성하라. epsilon 비교 함수를 직접 만들어 각 비교 결과도 출력하라.

5. **간단한 암호화/복호화**: XOR 비트 연산을 이용한 간단한 암호화 프로그램을 작성하라. 정수 값과 키(key)를 XOR하면 암호화되고, 같은 키로 다시 XOR하면 복호화된다는 원리를 활용하라.

## 프로젝트 과제

1. **다기능 계산기**: 두 수를 입력받고 연산자(+, -, *, /, %)를 선택하여 계산하는 프로그램을 작성하라. 정수 모드와 실수 모드를 지원하고, 0으로 나누기 검사, 오버플로우 감지(결과가 `math.MaxInt64`를 초과하는지), 실수 비교 시 epsilon 처리를 모두 포함하라. 연산 결과를 10진수, 2진수, 16진수로 출력하는 옵션도 제공하라.

2. **성적 등급 판정 시스템**: 학생의 이름과 국어, 영어, 수학 점수를 입력받아 총점, 평균을 계산하고, 평균에 따라 등급(A: 90이상, B: 80이상, C: 70이상, D: 60이상, F: 60미만)을 판정하는 프로그램을 작성하라. 산술 연산자, 비교 연산자, 논리 연산자를 종합적으로 활용하라. 결과를 보기 좋은 표 형태로 출력하라.
