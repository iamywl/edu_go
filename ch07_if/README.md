## 7.1 if문 기본 사용법

`if`문은 조건이 참(`true`)일 때 코드를 실행하는 조건 분기문이다. 프로그램의 흐름을 제어하는 가장 기본적인 도구이다.

```go
age := 20

if age >= 18 {
    fmt.Println("성인입니다")
}
```

Go의 `if`문은 다른 언어와 다르게 **조건에 괄호`()`를 쓰지 않는다**. 중괄호 `{}`는 반드시 필요하며, 본문이 한 줄이더라도 생략할 수 없다. 이는 코드 스타일의 일관성을 강제하기 위한 Go의 설계 철학이다.

```go
// Go에서는 이렇게 쓸 수 없다 (컴파일 에러)
if age >= 18
    fmt.Println("성인입니다")
```

### if-else

```go
score := 70

if score >= 60 {
    fmt.Println("합격")
} else {
    fmt.Println("불합격")
}
```

`else` 블록은 조건이 거짓(`false`)일 때 실행된다. `else`는 선택 사항이며, 필요할 때만 작성하면 된다.

### if-else if-else

```go
score := 85

if score >= 90 {
    fmt.Println("A")
} else if score >= 80 {
    fmt.Println("B")
} else if score >= 70 {
    fmt.Println("C")
} else {
    fmt.Println("F")
}
```

`else if`는 여러 개를 연결할 수 있다. 위에서부터 순서대로 조건을 검사하며, 처음으로 참이 되는 블록 하나만 실행된다. 따라서 조건의 순서가 중요하다.

> **주의:** `else`와 `else if`는 반드시 닫는 중괄호 `}`와 같은 줄에 있어야 한다. 이는 Go의 세미콜론 자동 삽입 규칙 때문이다. Go 컴파일러는 줄 끝에 특정 토큰이 오면 자동으로 세미콜론을 삽입하는데, `}`가 줄 끝에 오면 세미콜론이 삽입되어 `else`가 별도의 문장으로 인식된다.

```go
// 올바른 형태
if x > 0 {
    // ...
} else {
    // ...
}

// 컴파일 에러! (else가 다음 줄에 있으면 안 됨)
if x > 0 {
    // ...
}
else {
    // ...
}
```

## 7.2 그리고 `&&`, 또는 `||`

논리 연산자를 사용하면 여러 조건을 조합할 수 있다. Go에서 제공하는 논리 연산자는 세 가지이다.

### `&&` (AND) - 모든 조건이 참이어야 참

```go
age := 25
hasID := true

if age >= 18 && hasID {
    fmt.Println("입장 가능")
}
```

`&&`는 **단축 평가(short-circuit evaluation)**를 수행한다. 왼쪽 조건이 `false`이면 오른쪽 조건은 검사하지 않는다. 이 특성을 활용하면 안전한 코드를 작성할 수 있다.

```go
// arr이 nil이 아닐 때만 len을 검사한다
if arr != nil && len(arr) > 0 {
    fmt.Println(arr[0])
}
```

### `||` (OR) - 하나라도 참이면 참

```go
day := "토요일"

if day == "토요일" || day == "일요일" {
    fmt.Println("주말입니다!")
}
```

`||`도 단축 평가를 수행한다. 왼쪽 조건이 `true`이면 오른쪽 조건은 검사하지 않는다.

### `!` (NOT) - 조건을 반전

```go
isRaining := false

if !isRaining {
    fmt.Println("우산이 필요 없습니다")
}
```

`!` 연산자는 `true`를 `false`로, `false`를 `true`로 뒤집는다.

### 조합

여러 논리 연산자를 함께 사용할 수 있다. `&&`가 `||`보다 우선순위가 높다. 복잡한 조건은 괄호를 사용하여 의도를 명확히 하는 것이 좋다.

```go
temp := 25
humid := 60

if temp >= 20 && temp <= 28 && humid < 70 {
    fmt.Println("쾌적한 날씨입니다")
}

// 괄호로 우선순위를 명확히 표현
if (age >= 18 && hasID) || isVIP {
    fmt.Println("입장 가능")
}
```

## 7.3 중첩 if

`if`문 안에 `if`문을 넣을 수 있다. 하지만 너무 깊은 중첩은 가독성을 크게 해친다. 일반적으로 3단계 이상의 중첩은 피하는 것이 좋다.

```go
age := 20
hasTicket := true

if age >= 18 {
    if hasTicket {
        fmt.Println("입장 가능")
    } else {
        fmt.Println("티켓을 구매하세요")
    }
} else {
    fmt.Println("미성년자는 입장 불가")
}
```

**팁:** Go에서는 중첩을 줄이기 위해 **"빠른 반환(early return)"** 패턴을 강하게 선호한다. 이 패턴은 조건이 맞지 않을 때 먼저 반환하여, 정상 경로의 코드가 들여쓰기 없이 진행되도록 한다. Go 공식 코드 리뷰 가이드에서도 이 스타일을 권장한다.

```go
func canEnter(age int, hasTicket bool) string {
    if age < 18 {
        return "미성년자는 입장 불가"
    }
    if !hasTicket {
        return "티켓을 구매하세요"
    }
    return "입장 가능"
}
```

중첩된 버전과 비교하면, early return 패턴이 코드의 의도를 더 명확하게 전달하는 것을 알 수 있다. 에러나 예외 상황을 먼저 처리하고, 정상 흐름이 함수의 주된 경로가 되도록 하는 것이 핵심이다.

## 7.4 if 초기문; 조건문

Go만의 특별한 문법이다. `if`문에서 **조건을 검사하기 전에 짧은 초기화 문장**을 쓸 수 있다. 초기문에서 선언된 변수는 `if` 블록(및 연결된 `else if`, `else` 블록) 안에서만 유효하다.

```go
if 초기문; 조건문 {
    // ...
}
```

### 예제

```go
if num := 10; num%2 == 0 {
    fmt.Println(num, "은 짝수입니다")
}
// 여기서 num은 사용 불가 (if 블록 안에서만 유효)
```

이 문법의 핵심 장점은 **변수의 스코프를 최소화**할 수 있다는 것이다. 변수가 필요한 블록 안에서만 존재하므로, 바깥 스코프를 오염시키지 않는다.

### 실전 활용 - 에러 처리

```go
if err := doSomething(); err != nil {
    fmt.Println("에러 발생:", err)
}
```

이 패턴은 Go 코드에서 가장 자주 등장하는 관용구 중 하나이다. `err` 변수의 범위가 `if` 블록 안으로 제한되어 깔끔하다. 에러가 없을 때 `err` 변수가 불필요하게 남아있지 않는다.

### 함수 반환값과 함께

```go
if result, err := divide(10, 3); err != nil {
    fmt.Println("에러:", err)
} else {
    fmt.Println("결과:", result)
}
```

초기문에서 선언한 변수는 `else` 블록에서도 접근할 수 있다. 이는 `if`, `else if`, `else`가 하나의 연결된 블록으로 취급되기 때문이다.

### 주의사항

초기문에서 선언한 변수는 if 블록 바깥에서 사용할 수 없으므로, 블록 바깥에서도 해당 값이 필요하다면 초기문을 사용하지 말고 일반적인 변수 선언을 사용해야 한다.

```go
// 블록 바깥에서도 result가 필요한 경우
result, err := divide(10, 3)
if err != nil {
    fmt.Println("에러:", err)
    return
}
fmt.Println("결과:", result) // result를 계속 사용 가능
```

## 핵심 요약

| 항목 | 설명 |
|------|------|
| `if 조건 { }` | 기본 if문이다 (괄호 없음!) |
| `else` / `else if` | 닫는 `}`와 같은 줄에 작성해야 한다 |
| `&&`, `\|\|`, `!` | 논리 AND, OR, NOT 연산자이다 |
| `if 초기문; 조건문` | Go 특유의 초기문 패턴이다 |
| 빠른 반환 | 중첩 if 대신 early return을 선호한다 |
| 단축 평가 | `&&`와 `\|\|`는 왼쪽 결과에 따라 오른쪽을 건너뛸 수 있다 |

## 연습문제

1. 사용자의 점수(0~100)를 변수로 선언하고, 90 이상이면 "A", 80 이상이면 "B", 70 이상이면 "C", 60 이상이면 "D", 나머지는 "F"를 출력하는 프로그램을 작성하라.

2. 년도를 변수로 선언하고 윤년인지 판별하는 프로그램을 작성하라. (윤년 조건: 4의 배수이면서 100의 배수가 아닌 해, 또는 400의 배수인 해)

3. `if 초기문; 조건문` 패턴을 사용하여, 문자열의 길이가 5 이상이면 "긴 문자열", 아니면 "짧은 문자열"을 출력하는 코드를 작성하라.

4. 세 개의 정수 변수를 선언하고, 가장 큰 값을 출력하는 프로그램을 작성하라. `if-else if-else`를 사용하라.

5. 단축 평가(short-circuit evaluation)를 활용하는 예제를 작성하라. `&&` 연산에서 왼쪽 조건이 `false`일 때 오른쪽 조건이 실행되지 않는 것을 확인할 수 있도록, 오른쪽에 함수 호출을 배치하라.

6. 다음 조건을 `if`문으로 구현하라: 나이가 13세 이상 19세 이하이면 "청소년", 20세 이상 64세 이하이면 "성인", 65세 이상이면 "노인", 나머지는 "어린이"를 출력하라.

7. `if 초기문; 조건문` 패턴을 사용하여 정수를 2로 나눈 몫과 나머지를 구하고, 나머지가 0이면 "짝수", 아니면 "홀수"를 출력하는 코드를 작성하라.

8. 빠른 반환(early return) 패턴을 사용하여 사용자 인증 함수를 작성하라. 조건: 사용자 이름이 빈 문자열이면 에러, 비밀번호가 8자 미만이면 에러, 비밀번호가 "password"이면 에러, 모두 통과하면 "인증 성공"을 반환하라.

9. 아래 코드의 출력 결과를 예측하고, 그 이유를 설명하라.
   ```go
   x := 10
   if x := 5; x > 3 {
       fmt.Println("A:", x)
   }
   fmt.Println("B:", x)
   ```

10. 논리 연산자 우선순위를 확인하는 문제이다. 다음 조건식의 결과를 예측하라.
    ```go
    a, b, c := true, false, true
    fmt.Println(a || b && c)   // 결과는?
    fmt.Println((a || b) && c) // 결과는?
    ```

## 구현 과제

### 과제 1: 간단한 성적 처리 프로그램
학생의 국어, 영어, 수학 점수를 변수로 선언하라. 각 과목이 0~100 범위인지 검증하고, 평균을 구한 뒤 학점(A/B/C/D/F)을 출력하라. 만약 한 과목이라도 40점 미만이면 과락으로 "F"를 부여하라.

### 과제 2: 택시 요금 계산기
거리(km)와 시간대(0~23시)를 변수로 선언하라. 기본요금(2km까지 4800원), 추가요금(km당 1000원)을 계산하고, 심야 시간(22시~06시)이면 요금에 20%를 할증하라. `if 초기문` 패턴을 적절히 활용하라.

### 과제 3: 삼각형 판별기
세 변의 길이를 입력받아 삼각형이 될 수 있는지 판별하는 프로그램을 작성하라. (삼각형 조건: 가장 긴 변이 나머지 두 변의 합보다 작아야 한다) 삼각형이 가능하면, 정삼각형/이등변삼각형/직각삼각형/일반삼각형인지 분류하여 출력하라.

### 과제 4: BMI 판정 프로그램
키(cm)와 몸무게(kg)를 변수로 선언하고, BMI를 계산하라. BMI에 따라 저체중/정상/과체중/비만을 판별하되, early return 패턴을 사용하는 함수로 구현하라. 입력값이 유효한 범위인지도 검증하라. (키: 50~250cm, 몸무게: 10~300kg)

### 과제 5: 날짜 유효성 검사기
년, 월, 일을 변수로 선언하고, 해당 날짜가 유효한지 검사하는 프로그램을 작성하라. 각 월의 일수(28/29/30/31)를 올바르게 처리하고, 2월의 경우 윤년 여부에 따라 29일까지 허용하라.

## 프로젝트 과제

### 프로젝트 1: 놀이공원 입장 시스템
놀이공원의 입장 및 이용 규칙을 `if`문으로 구현하라. 다음 조건들을 모두 처리하라:
- 기본 입장: 4세 이상 입장 가능, 4세 미만은 보호자 동반 시 입장 가능
- 할인: 어린이(4~12세) 50%, 청소년(13~18세) 30%, 경로(65세 이상) 40%
- 놀이기구 탑승: 키 120cm 이상만 탑승 가능, 일부 기구는 140cm 이상
- 시간대별 요금: 오후 4시 이후 입장 시 30% 할인 (타 할인과 중복 적용)
- 입장 정보(나이, 키, 보호자 동반 여부, 시간)를 변수로 설정하고, 최종 입장 가능 여부, 요금, 탑승 가능 기구를 출력하라.

### 프로젝트 2: 간이 로그인 시스템
사용자 이름과 비밀번호를 검사하는 로그인 시스템을 구현하라. 다음 기능을 포함하라:
- 사용자 정보는 상수 또는 변수로 미리 정의 (예: admin/pass1234)
- 로그인 시도 횟수를 추적하여, 3회 실패 시 "계정 잠금" 메시지 출력
- 비밀번호 강도 검사: 길이 8자 이상, 숫자 포함 여부 확인 (문자열을 순회하며 확인)
- 로그인 성공/실패/계정 잠금 상태를 구분하여 메시지 출력
- `if 초기문` 패턴과 early return 패턴을 적극 활용하라.
