## 2.1 변수란?

변수(Variable)는 **값을 저장하는 메모리 공간에 이름을 붙인 것**이다.

```
메모리 주소    값        변수 이름
0x00A1       25        age
0x00B2       "홍길동"   name
0x00C3       3.14      pi
```

- 변수를 만들면 컴퓨터 메모리(RAM)에 일정 크기의 공간이 확보된다.
- 그 공간에 이름을 붙여 값을 저장하고 꺼내 쓸 수 있다.
- 이름이 "변수"인 이유는 저장된 값이 **변할 수 있기** 때문이다.

변수를 이해하려면 메모리의 동작 방식을 간단히 알아야 한다. 컴퓨터 메모리는 바이트(byte) 단위의 연속된 공간이며, 각 바이트에는 고유한 주소가 있다. `int` 타입 변수를 선언하면 해당 타입의 크기(보통 8바이트)만큼 메모리가 할당되고, 그 시작 주소에 변수 이름이 연결되는 것이다.

## 2.2 변수 선언 (var 키워드)

Go에서 변수를 선언하는 기본 형태:

```go
var 변수이름 타입
```

### 선언과 초기화

```go
// 선언만 (기본값으로 초기화됨)
var age int

// 선언과 동시에 초기화
var name string = "홍길동"

// 타입 추론 (값에서 타입을 자동으로 결정)
var pi = 3.14  // float64로 추론된다
```

타입 추론은 컴파일러가 대입되는 값을 보고 자동으로 타입을 결정하는 기능이다. 타입을 명시하지 않아도 컴파일러가 올바른 타입을 유추하므로 코드가 간결해진다. 단, 정수 리터럴은 `int`로, 실수 리터럴은 `float64`로 추론된다는 점을 알아야 한다.

### 변수 이름 규칙

| 규칙 | 예시 |
|------|------|
| 문자, 숫자, `_` 사용 가능 | `userName`, `count1`, `max_value` |
| 숫자로 시작 불가 | `1name` (에러) |
| 대소문자 구분 | `Name`과 `name`은 다른 변수이다 |
| 예약어 사용 불가 | `func`, `var`, `if` 등은 변수 이름으로 사용할 수 없다 |
| 대문자 시작 = 외부 공개 | `Name` (exported), `name` (unexported) |

Go에는 25개의 예약어(keyword)가 있다: `break`, `case`, `chan`, `const`, `continue`, `default`, `defer`, `else`, `fallthrough`, `for`, `func`, `go`, `goto`, `if`, `import`, `interface`, `map`, `package`, `range`, `return`, `select`, `struct`, `switch`, `type`, `var`. 이 단어들은 변수 이름으로 사용할 수 없다.

### Go의 이름 관례 (Naming Convention)

Go에서는 **카멜 케이스(camelCase)**를 사용한다:

```go
var userName string    // 좋은 예 (camelCase) - 패키지 내부에서만 사용
var user_name string   // Go에서는 권장하지 않는다 (snake_case)
var UserName string    // 패키지 외부에 공개할 때 사용한다 (PascalCase)
```

Go 커뮤니티에서는 변수 이름을 짧고 명확하게 짓는 것을 선호한다. 예를 들어 루프 변수에는 `index` 대신 `i`를, 수신자(receiver)에는 타입의 첫 글자를 사용하는 것이 관례이다. 단, 이름이 사용되는 범위가 넓을수록 더 서술적인 이름을 사용해야 한다.

## 2.3 변수에 대해 더 알아보기

### 기본값 (Zero Value)

Go에서는 변수를 초기화하지 않으면 **타입에 따른 기본값(zero value)**이 자동으로 할당된다. 이것은 Go의 중요한 설계 원칙 중 하나로, 초기화되지 않은 변수에 접근하여 예측 불가능한 동작이 발생하는 것을 방지한다. C/C++에서는 초기화하지 않은 변수에 쓰레기 값이 들어있어 버그의 원인이 되곤 하는데, Go에서는 이 문제가 원천적으로 차단된다.

| 타입 | 기본값 |
|------|--------|
| `bool` | `false` |
| `int`, `int8`, `int16`, `int32`, `int64` | `0` |
| `uint`, `uint8`, `uint16`, `uint32`, `uint64` | `0` |
| `float32`, `float64` | `0.0` |
| `string` | `""` (빈 문자열) |
| 포인터, 슬라이스, 맵, 채널, 함수, 인터페이스 | `nil` |

### Go의 기본 타입 종류

#### 정수 타입

| 타입 | 크기 | 범위 |
|------|------|------|
| `int8` | 1바이트 | -128 ~ 127 |
| `int16` | 2바이트 | -32,768 ~ 32,767 |
| `int32` | 4바이트 | -21억 ~ 21억 |
| `int64` | 8바이트 | -922경 ~ 922경 |
| `int` | 플랫폼 의존 | 32비트 또는 64비트 |

`int`는 플랫폼에 따라 크기가 달라지는데, 현대의 64비트 시스템에서는 보통 64비트(8바이트)이다. 특별한 이유가 없다면 정수에는 `int`를 사용하는 것이 관례이다. 특정 크기가 필요한 경우(예: 네트워크 프로토콜, 파일 포맷 처리)에만 `int8`, `int16` 등 크기가 명시된 타입을 사용한다.

#### 부호 없는 정수 타입

| 타입 | 크기 | 범위 |
|------|------|------|
| `uint8` (= `byte`) | 1바이트 | 0 ~ 255 |
| `uint16` | 2바이트 | 0 ~ 65,535 |
| `uint32` | 4바이트 | 0 ~ 42억 |
| `uint64` | 8바이트 | 0 ~ 1844경 |

부호 없는 정수(unsigned integer)는 음수를 표현할 수 없는 대신, 같은 크기에서 양수 범위가 두 배로 넓다. 음수가 될 수 없는 값(배열 인덱스, 길이, 용량 등)에 적합하다.

#### 실수 타입

| 타입 | 크기 | 정밀도 |
|------|------|--------|
| `float32` | 4바이트 | 소수점 이하 약 7자리 |
| `float64` | 8바이트 | 소수점 이하 약 15자리 |

특별한 이유가 없다면 `float64`를 사용하는 것이 권장된다. `float32`는 정밀도가 낮아서 계산이 누적될수록 오차가 커질 수 있다.

#### 기타 타입

| 타입 | 설명 |
|------|------|
| `bool` | 참(`true`) 또는 거짓(`false`)을 나타낸다 |
| `string` | 문자열이다 (UTF-8 인코딩). 불변(immutable)이다 |
| `byte` | `uint8`의 별칭이다. 바이트 데이터를 다룰 때 사용한다 |
| `rune` | `int32`의 별칭이다 (유니코드 코드 포인트). 한글, 이모지 등 다국어 문자를 다룰 때 사용한다 |

Go에서 `string`은 불변(immutable)이다. 문자열을 변경하면 새로운 문자열이 생성되는 것이지, 원래 문자열이 수정되는 것이 아니다.

## 2.4 변수 선언의 다른 형태

### 짧은 선언 (Short Variable Declaration) `:=`

함수 안에서만 사용할 수 있는 간편한 선언 방법이다:

```go
func main() {
    name := "Go"       // var name string = "Go"와 동일하다
    age := 15          // var age int = 15와 동일하다
    pi := 3.14         // var pi float64 = 3.14와 동일하다
    isGood := true     // var isGood bool = true와 동일하다
}
```

> **주의**: `:=`는 **함수 안에서만** 사용 가능하다. 패키지 레벨(함수 바깥)에서는 반드시 `var`를 사용해야 한다.

`:=`를 사용할 때 주의할 점이 하나 더 있다. 이미 선언된 변수에 `:=`를 다시 사용하면 컴파일 에러가 발생한다. 단, 여러 변수를 동시에 선언할 때 최소 하나의 새 변수가 있으면 허용된다:

```go
x := 10
// x := 20         // 에러! x는 이미 선언되었다
x, y := 20, 30     // 허용. y가 새 변수이기 때문이다
```

### 여러 변수 동시 선언

```go
// var 블록 사용
var (
    name   string = "홍길동"
    age    int    = 25
    height float64 = 175.5
)

// 같은 타입의 여러 변수
var x, y, z int

// 짧은 선언으로 여러 변수
a, b, c := 1, "hello", true
```

`var` 블록은 패키지 레벨에서 여러 변수를 선언할 때 특히 유용하다. 관련된 변수를 하나의 블록으로 묶어 코드의 가독성을 높일 수 있다.

### 상수 (const)

값이 변하지 않는 변수를 **상수**라고 한다:

```go
const Pi = 3.14159265358979
const MaxRetry = 3

// const 블록
const (
    StatusOK    = 200
    StatusError = 500
)

// iota: 자동 증가 상수
const (
    Sunday = iota  // 0
    Monday         // 1
    Tuesday        // 2
    Wednesday      // 3
)
```

`iota`는 Go만의 독특한 기능으로, `const` 블록 안에서 자동으로 0부터 1씩 증가하는 값을 생성한다. 열거형(enum)이 별도로 없는 Go에서 `iota`가 그 역할을 대신한다.

`iota`는 더 복잡한 패턴으로도 사용할 수 있다:

```go
// 비트 플래그
const (
    ReadPermission  = 1 << iota  // 1 (001)
    WritePermission              // 2 (010)
    ExecPermission               // 4 (100)
)

// 시작값 건너뛰기
const (
    _ = iota   // 0을 건너뛴다
    One        // 1
    Two        // 2
    Three      // 3
)
```

상수는 컴파일 시점에 값이 결정되므로, 함수 호출의 결과를 상수에 할당할 수 없다. 오직 리터럴 값이나 컴파일 시점에 계산 가능한 표현식만 사용할 수 있다.

## 2.5 타입 변환

Go는 **암시적 타입 변환을 허용하지 않는다**. 반드시 명시적으로 변환해야 한다. 이것은 Go의 타입 안전성을 강화하는 설계 결정이다. 다른 언어에서 암시적 변환으로 인해 발생하는 미묘한 버그를 원천적으로 방지할 수 있다.

```go
var i int = 42
var f float64 = float64(i)    // int → float64
var u uint = uint(f)          // float64 → uint

// 주의: 데이터 손실이 발생할 수 있다
var bigNum int64 = 300
var smallNum int8 = int8(bigNum)  // 300은 int8 범위(-128~127)를 초과 → 44가 된다

// 문자열 변환
import "strconv"
str := strconv.Itoa(42)        // int → string: "42"
num, err := strconv.Atoi("42") // string → int: 42
```

> **중요**: Go에서는 `int`와 `int64`도 서로 다른 타입이다! 변환이 필요하다.

`string()`으로 정수를 변환하면 해당 숫자에 대응하는 유니코드 문자가 되므로 주의해야 한다:

```go
s := string(65)    // "A" (65는 'A'의 유니코드 코드 포인트)
// 숫자를 문자열로 변환하려면 반드시 strconv.Itoa()를 사용한다
s2 := strconv.Itoa(65) // "65"
```

## 2.6 변수의 범위 (스코프)

변수의 **스코프(scope)**란 변수가 유효한 범위를 말한다. Go에서 스코프는 중괄호 `{}`로 결정된다.

### 패키지 레벨 변수

```go
package main

var globalVar = "패키지 전체에서 접근 가능" // 패키지 레벨

func main() {
    fmt.Println(globalVar) // 접근 가능
}

func other() {
    fmt.Println(globalVar) // 접근 가능
}
```

패키지 레벨 변수는 프로그램이 시작될 때 초기화되며, 같은 패키지 내의 모든 함수에서 접근할 수 있다. 대문자로 시작하면 다른 패키지에서도 접근 가능하다.

### 함수 레벨 변수

```go
func main() {
    localVar := "main 함수 안에서만 접근 가능"
    fmt.Println(localVar)  // 접근 가능
}

func other() {
    // fmt.Println(localVar) // 컴파일 에러! localVar에 접근 불가하다
}
```

### 블록 레벨 변수

```go
func main() {
    x := 10

    if true {
        y := 20          // if 블록 안에서만 유효하다
        fmt.Println(x)   // 접근 가능 (바깥 변수)
        fmt.Println(y)   // 접근 가능
    }

    fmt.Println(x)       // 접근 가능
    // fmt.Println(y)    // 컴파일 에러! y는 if 블록 안에서만 유효하다
}
```

### 섀도잉 (Shadowing)

안쪽 스코프에서 바깥 변수와 같은 이름을 사용하면, 안쪽 변수가 바깥 변수를 **가린다**:

```go
x := 10
fmt.Println(x)    // 10

{
    x := 20       // 새로운 x (바깥의 x를 가린다)
    fmt.Println(x) // 20
}

fmt.Println(x)    // 10 (바깥의 x는 변하지 않는다)
```

섀도잉은 의도치 않은 버그의 원인이 될 수 있으므로 주의해야 한다. 특히 `:=`를 사용할 때 기존 변수를 재사용하려는 의도인데 실수로 새 변수를 선언하는 경우가 흔하다. `go vet`이나 `shadow` 린터를 사용하면 이런 문제를 사전에 감지할 수 있다.

## 2.7 숫자 표현

Go에서는 다양한 진법으로 숫자를 표현할 수 있다.

```go
// 10진수 (기본)
decimal := 42

// 2진수 (0b 접두사)
binary := 0b101010  // 42

// 8진수 (0o 접두사)
octal := 0o52       // 42

// 16진수 (0x 접두사)
hex := 0x2A         // 42

// 가독성을 위한 밑줄(_) 사용
billion := 1_000_000_000
hexColor := 0xFF_AA_BB
```

밑줄(`_`)은 숫자 리터럴의 가독성을 높이기 위한 순수한 시각적 구분자이다. 컴파일 시 무시되므로 어떤 위치에든 넣을 수 있다. 큰 숫자를 다룰 때 천 단위로 밑줄을 넣으면 한눈에 값을 파악하기 쉬워진다.

### 출력 형식

```go
num := 42
fmt.Printf("10진수: %d\n", num)   // 42
fmt.Printf("2진수:  %b\n", num)   // 101010
fmt.Printf("8진수:  %o\n", num)   // 52
fmt.Printf("16진수: %x\n", num)   // 2a
fmt.Printf("16진수: %X\n", num)   // 2A
```

## 핵심 요약

| 항목 | 설명 |
|------|------|
| `var x int` | var 키워드로 변수를 선언한다 |
| `x := 10` | 짧은 선언이다 (함수 안에서만 가능) |
| 기본값 | int=0, string="", bool=false이다 |
| 타입 변환 | `float64(i)` 형태로 명시적 변환이 필수이다 |
| 스코프 | 패키지 > 함수 > 블록 순으로 범위가 좁아진다 |
| 상수 | `const`로 선언하며, 값을 변경할 수 없다 |

## 연습문제

1. 자신의 이름, 나이, 키를 각각 적절한 타입의 변수에 저장하고 출력하는 프로그램을 작성하라.
2. `int` 타입 변수에 저장된 값을 `float64`로 변환하여 나눗셈의 결과를 소수점까지 출력하라.
3. 다음 코드의 출력 결과를 예측해 보라:
   ```go
   x := 10
   {
       x := 20
       fmt.Println(x)
   }
   fmt.Println(x)
   ```
4. 255를 2진수, 8진수, 16진수로 출력하는 프로그램을 작성하라.
5. `const`와 `iota`를 사용하여 계절(봄=0, 여름=1, 가을=2, 겨울=3)을 정의하는 상수를 만들어 보라.

## 구현 과제

1. **개인 정보 카드 프로그램**: 이름(`string`), 나이(`int`), 키(`float64`), 결혼 여부(`bool`)를 변수로 선언하고, `fmt.Printf`를 사용하여 보기 좋게 출력하는 프로그램을 작성하라. 각 변수의 타입도 `%T` 서식 지정자로 함께 출력하라.

2. **온도 변환기**: 섭씨 온도를 `float64` 변수에 저장하고, 이를 화씨(F = C*9/5 + 32)와 켈빈(K = C + 273.15)으로 변환하여 출력하는 프로그램을 작성하라. 모든 결과를 소수점 2자리까지 출력하라.

3. **데이터 타입 크기 탐험기**: `unsafe.Sizeof()` 함수를 사용하여 `int8`, `int16`, `int32`, `int64`, `float32`, `float64`, `bool`, `string` 타입 변수의 메모리 크기(바이트)를 출력하는 프로그램을 작성하라. (`import "unsafe"` 필요)

4. **진법 변환 출력기**: 0부터 15까지의 숫자를 10진수, 2진수, 8진수, 16진수로 정렬하여 표 형태로 출력하는 프로그램을 작성하라. `fmt.Printf`의 너비 지정 기능을 활용하라.

5. **상수 활용 프로그램**: `const`와 `iota`를 사용하여 HTTP 상태 코드(200, 201, 301, 400, 401, 403, 404, 500)를 정의하고, 각 코드의 이름(OK, Created, Moved Permanently 등)을 출력하는 프로그램을 작성하라. `iota`만으로는 불연속적인 값을 표현하기 어려우므로, `const` 블록에 직접 값을 할당하는 방식과 `iota`를 활용하는 방식의 차이를 비교하라.
