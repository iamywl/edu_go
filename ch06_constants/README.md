## 6.1 상수 선언

상수(constant)는 **한 번 정해지면 변하지 않는 값**이다. Go에서는 `const` 키워드로 선언한다.

```go
const Pi = 3.14159
const Greeting = "안녕하세요"
```

변수(`var`)와 달리 상수는 선언 후 값을 변경할 수 없다. 컴파일 시점에 값이 결정되며, 런타임에 변경하려고 하면 컴파일 에러가 발생한다.

```go
const MaxRetry = 3
MaxRetry = 5 // 컴파일 에러! 상수는 변경 불가
```

여러 상수를 한꺼번에 선언할 수도 있다. 괄호로 묶어서 선언하면 `const` 키워드를 반복하지 않아도 된다.

```go
const (
    Name    = "Go"
    Version = 1.21
    IsOpen  = true
)
```

상수에 사용할 수 있는 값은 **컴파일 타임에 결정 가능한 값**으로 제한된다. 즉, 함수 호출 결과나 런타임에 결정되는 값은 상수에 대입할 수 없다.

```go
const A = 10            // OK: 리터럴
const B = 20 + 30       // OK: 컴파일 타임 연산
const C = math.Pi       // OK: math.Pi도 상수
const D = len("hello")  // OK: 문자열 리터럴의 len은 컴파일 타임에 계산 가능

const E = rand.Intn(10) // 컴파일 에러! 런타임 함수 호출 결과는 상수가 될 수 없다
```

## 6.2 상수는 언제 사용하나?

상수를 사용하는 대표적인 경우는 다음과 같다.

### 변하지 않는 값을 명확히 표현할 때

```go
const TaxRate = 0.1       // 세율 10%
const MaxStudents = 30     // 최대 학생 수
```

숫자 `0.1`이나 `30`을 코드 곳곳에 그대로 쓰면 나중에 그 숫자가 무엇을 의미하는지 알기 어렵다. 이런 숫자를 **매직 넘버(magic number)**라고 부르며, 코드 유지보수에 큰 장애물이 된다. 상수로 이름을 붙이면 **코드 가독성**이 크게 향상된다.

### 실수로 값을 바꾸는 것을 방지할 때

변수로 선언하면 누군가 실수로 값을 바꿀 수 있다. 상수로 선언하면 컴파일러가 변경을 막아준다. 물리 법칙이나 수학 상수처럼 절대 변하면 안 되는 값에 적합하다.

```go
const GravityAccel = 9.8   // 중력가속도 - 절대 바뀌면 안 됨
const SpeedOfLight = 299792458 // 광속 (m/s)
```

### 여러 곳에서 공통으로 사용하는 값을 한 곳에서 관리할 때

```go
const (
    DBHost = "localhost"
    DBPort = 5432
    DBName = "myapp"
)
```

이렇게 선언하면 값을 변경해야 할 때 한 곳만 수정하면 된다. 코드 전체에 흩어진 리터럴을 하나하나 찾아 수정하는 수고를 덜 수 있다.

## 6.3 타입 없는 상수 (Untyped Constant)

Go의 상수에는 특별한 점이 있다. **타입을 지정하지 않은 상수**는 사용하는 곳에 따라 자동으로 타입이 결정된다. 이를 "타입 없는 상수(untyped constant)"라고 부른다.

```go
const Big = 1000000000   // 타입 없는 상수

var a int32 = Big        // int32로 사용
var b int64 = Big        // int64로 사용
var c float64 = Big      // float64로 사용
```

타입 없는 상수는 내부적으로 매우 높은 정밀도를 가진다. 예를 들어 타입 없는 정수 상수는 최소 256비트의 정밀도를 보장한다. 이 덕분에 일반 변수로는 표현할 수 없는 큰 중간 계산도 가능하다.

```go
const Huge = 1 << 200  // 2^200 - 변수로는 저장 불가능하지만 상수로는 가능
const Ratio = Huge / (1 << 100)  // 중간 계산에 사용 가능
```

타입을 명시하면 해당 타입에 고정된다.

```go
const TypedBig int32 = 1000000000

var a int32 = TypedBig    // OK
var b int64 = TypedBig    // 컴파일 에러! int32를 int64에 대입 불가
```

타입 없는 상수는 리터럴처럼 유연하게 동작하므로, **특별한 이유가 없다면 상수에 타입을 지정하지 않는 것**이 Go의 관용적인 스타일이다.

### 타입 없는 상수의 기본 타입

타입 없는 상수도 "기본 타입(default type)"을 가지고 있다. 타입 추론이 필요한 상황에서 이 기본 타입이 사용된다.

| 상수 종류 | 기본 타입 | 예시 |
|-----------|----------|------|
| 정수 | `int` | `const A = 42` |
| 실수 | `float64` | `const B = 3.14` |
| 복소수 | `complex128` | `const C = 1 + 2i` |
| 룬(문자) | `rune` | `const D = 'A'` |
| 문자열 | `string` | `const E = "hello"` |
| 불리언 | `bool` | `const F = true` |

```go
const X = 10
y := X  // y의 타입은 int (정수 상수의 기본 타입)
```

## 6.4 상수와 리터럴 - iota를 이용한 열거값

`iota`는 `const` 블록 안에서 0부터 시작하여 줄마다 1씩 증가하는 특수한 값이다. 열거형(enum)을 만들 때 매우 유용하다.

### 기본 사용법

```go
const (
    Sunday    = iota   // 0
    Monday             // 1
    Tuesday            // 2
    Wednesday          // 3
    Thursday           // 4
    Friday             // 5
    Saturday           // 6
)
```

첫 번째 줄에 `iota`를 쓰면, 나머지 줄은 자동으로 같은 패턴을 따른다. 이는 Go 컴파일러가 `const` 블록 안에서 이전 줄의 표현식을 반복 적용하기 때문이다.

### 1부터 시작하기

```go
const (
    _       = iota   // 0 (빈 식별자로 건너뜀)
    Red              // 1
    Green            // 2
    Blue             // 3
)
```

빈 식별자 `_`를 사용하여 0을 건너뛰는 패턴은 실전에서 자주 사용된다. 0값은 보통 "미설정" 또는 "알 수 없음"을 나타내므로, 유효한 열거값을 1부터 시작하는 것이 안전하다.

### 비트 플래그 패턴

```go
const (
    ReadPerm   = 1 << iota  // 1  (0001)
    WritePerm               // 2  (0010)
    ExecPerm                // 4  (0100)
)
```

비트 플래그는 여러 값을 OR(`|`) 연산으로 조합할 수 있다는 장점이 있다. 파일 권한, 옵션 플래그 등에 자주 활용된다.

```go
permission := ReadPerm | ExecPerm  // 5 (0101) - 읽기+실행 권한
fmt.Println(permission & ReadPerm != 0)  // true - 읽기 권한 있음
fmt.Println(permission & WritePerm != 0) // false - 쓰기 권한 없음
```

### iota와 수식 활용

`iota`에 수식을 적용하면 다양한 패턴의 열거값을 만들 수 있다.

```go
// 바이트 단위 (KB, MB, GB ...)
const (
    _  = iota
    KB = 1 << (10 * iota)  // 1024
    MB                      // 1048576
    GB                      // 1073741824
    TB                      // 1099511627776
)
```

`iota`는 각 `const` 블록마다 0으로 리셋된다. 서로 다른 `const` 블록에서 `iota`를 사용하면 각각 독립적으로 동작한다.

## 핵심 요약

| 항목 | 설명 |
|------|------|
| `const` | 변하지 않는 값을 선언한다 |
| 타입 없는 상수 | 사용처에 따라 타입이 자동 결정된다 (유연하다) |
| 타입 있는 상수 | 지정한 타입에 고정된다 |
| `iota` | const 블록에서 0부터 1씩 증가하는 열거값을 생성한다 |
| 용도 | 코드 가독성 향상, 실수 방지, 열거값 정의에 사용한다 |

## 구현 과제

### 과제 1: 온도 변환기
섭씨-화씨 변환 공식에 필요한 값들을 상수로 정의하고, 섭씨 온도 배열(0, 20, 37, 100)을 화씨로 변환하여 출력하는 프로그램을 작성하라. (`F = C * 9/5 + 32`)

### 과제 2: 요일 출력기
`iota`를 사용하여 요일 상수(일요일~토요일)를 정의하고, 숫자를 입력하면 해당 요일 이름을 한국어로 출력하는 `dayName` 함수를 작성하라. 잘못된 숫자가 입력되면 "알 수 없는 요일"을 반환하도록 하라.

### 과제 3: 비트 플래그 권한 시스템
`iota`와 비트 연산을 활용하여 사용자 권한 시스템을 구현하라. 읽기, 쓰기, 실행, 관리자 권한을 비트 플래그로 정의하고, 권한을 추가/제거/확인하는 함수들(`addPermission`, `removePermission`, `hasPermission`)을 작성하라. 다양한 권한 조합을 테스트하여 결과를 출력하라.

### 과제 4: 데이터 크기 변환기
`iota`를 이용하여 바이트 단위(B, KB, MB, GB, TB)를 상수로 정의하고, 입력된 바이트 수를 가장 적절한 단위로 변환하여 출력하는 프로그램을 작성하라. 예: `1536` -> `"1.5 KB"`, `2621440` -> `"2.5 MB"`.

## 프로젝트 과제

### 프로젝트 1: 간이 계산기 상수 시스템
사칙연산 계산기를 만들되, 연산자를 `iota`로 정의한 상수로 표현하라. `Add`, `Sub`, `Mul`, `Div` 상수를 정의하고, `calculate(op int, a, b float64) (float64, error)` 함수를 구현하라. 0으로 나누는 경우의 에러 처리도 포함하라. 여러 연산을 수행하고 결과를 출력하는 `main` 함수를 작성하라.

### 프로젝트 2: 카드 게임 상수 정의
트럼프 카드의 무늬(스페이드, 하트, 다이아몬드, 클로버)와 숫자(A, 2~10, J, Q, K)를 `iota`와 상수를 활용하여 정의하라. 카드 한 장을 나타내는 값을 만들고(예: 무늬*13 + 숫자), 해당 값에서 무늬와 숫자를 역산하여 "스페이드 A", "하트 10" 등의 문자열로 출력하는 함수를 작성하라. 전체 52장의 카드를 생성하고 출력하는 프로그램을 완성하라.
