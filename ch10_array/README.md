## 10.1 배열

배열(array)은 **같은 타입**의 데이터를 **고정된 크기**로 저장하는 자료구조이다. 메모리에 **연속적으로** 저장되며, 프로그래밍에서 가장 기본적인 자료구조 중 하나이다.

```go
var scores [5]int  // int 5개를 저장하는 배열
```

Go 배열의 특징은 다음과 같다:
- **크기가 고정**이다. 선언 후 크기를 변경할 수 없다. 크기를 늘리거나 줄여야 한다면 슬라이스(slice)를 사용해야 한다.
- 크기가 **타입의 일부**이다. `[3]int`와 `[5]int`는 서로 다른 타입이다. 따라서 `[3]int` 변수에 `[5]int` 값을 대입할 수 없다.
- 선언 시 **기본값(zero value)**으로 초기화된다. (`int`는 0, `string`은 "", `bool`은 false)
- Go에서 배열은 **값 타입**이다. 대입이나 함수 전달 시 전체 복사가 일어난다.

```go
var a [3]int  // [0, 0, 0]
var b [5]int  // [0, 0, 0, 0, 0]
// a = b  // 컴파일 에러! [3]int와 [5]int는 다른 타입이다
```

## 10.2 배열 사용법

### 선언과 초기화

```go
// 선언만 (기본값으로 초기화)
var a [5]int  // [0, 0, 0, 0, 0]

// 선언과 동시에 초기화
b := [3]string{"사과", "바나나", "체리"}

// 크기를 자동 계산 (...)
c := [...]int{10, 20, 30, 40}  // 크기 4

// 특정 인덱스만 초기화
d := [5]int{1: 10, 3: 30}  // [0, 10, 0, 30, 0]
```

`[...]` 문법은 초기화 요소의 개수를 컴파일러가 세어서 크기를 결정한다. 배열 크기가 많은 경우 일일이 세는 수고를 덜 수 있다.

인덱스 지정 초기화에서는 지정하지 않은 인덱스는 기본값으로 채워진다. 마지막 인덱스를 큰 값으로 지정하면 그에 맞는 크기의 배열이 생성된다.

```go
e := [...]int{9: 100}  // 크기 10의 배열, e[9]만 100이고 나머지는 0
```

### 요소 접근과 수정

```go
arr := [5]int{10, 20, 30, 40, 50}

fmt.Println(arr[0])   // 10 (첫 번째 요소)
fmt.Println(arr[4])   // 50 (마지막 요소)

arr[2] = 99           // 세 번째 요소를 99로 변경
```

인덱스는 **0부터** 시작한다. 마지막 요소의 인덱스는 `len(arr) - 1`이다. 범위를 벗어나는 인덱스로 접근하면 **런타임 패닉(panic)**이 발생한다. 컴파일 시점에 상수 인덱스가 범위를 벗어나면 컴파일 에러가 발생한다.

```go
arr := [3]int{1, 2, 3}
// fmt.Println(arr[3])  // 컴파일 에러! 인덱스 3은 크기 3의 배열에서 벗어난다
// fmt.Println(arr[-1]) // 컴파일 에러! 음수 인덱스는 허용되지 않는다
```

### 배열 길이

```go
arr := [5]int{1, 2, 3, 4, 5}
fmt.Println(len(arr))  // 5
```

`len()` 함수는 배열의 크기를 반환한다. 배열의 크기는 컴파일 타임에 결정되므로, `len()`의 결과도 상수처럼 취급된다.

### for문으로 순회

```go
arr := [5]string{"Go", "는", "재미", "있", "다"}

// 인덱스로 순회
for i := 0; i < len(arr); i++ {
    fmt.Println(i, arr[i])
}

// range로 순회 (더 Go다운 방식)
for i, v := range arr {
    fmt.Println(i, v)
}

// 값만 필요할 때
for _, v := range arr {
    fmt.Print(v)
}
```

`range`를 사용하는 것이 Go에서 권장되는 방식이다. 인덱스 범위 초과 실수를 방지할 수 있고, 코드가 더 간결하다.

### 배열 비교

같은 타입, 같은 크기의 배열은 `==`로 비교할 수 있다. 모든 요소가 같으면 `true`를 반환한다.

```go
a := [3]int{1, 2, 3}
b := [3]int{1, 2, 3}
c := [3]int{1, 2, 4}

fmt.Println(a == b) // true
fmt.Println(a == c) // false
```

이는 Go 배열의 값 타입 특성 덕분이다. 슬라이스는 `==`로 비교할 수 없다는 점과 대조된다.

## 10.3 배열은 연속된 메모리

배열의 요소는 메모리에 **빈틈없이 연속**으로 저장된다. 이 특성이 배열의 성능상 이점을 제공한다.

```
[5]int 배열의 메모리 레이아웃 (64비트 시스템):

주소:    0x100  0x108  0x110  0x118  0x120
         +------+------+------+------+------+
값:      |  10  |  20  |  30  |  40  |  50  |
         +------+------+------+------+------+
인덱스:    [0]    [1]    [2]    [3]    [4]

각 int는 8바이트(64비트), 총 40바이트
```

연속 메모리의 장점은 다음과 같다:
- **빠른 접근**: 인덱스로 즉시 원하는 요소에 접근할 수 있다 (O(1)). 배열의 시작 주소에 `인덱스 * 요소크기`를 더하면 해당 요소의 주소가 된다.
- **캐시 효율**: CPU 캐시에 연속으로 올라가므로 순차 접근이 빠르다. 현대 CPU에서 캐시 효율은 성능에 매우 큰 영향을 미친다.
- **메모리 효율**: 포인터나 추가 메타데이터 없이 데이터만 저장하므로 메모리 오버헤드가 없다.

### 배열의 전체 크기 계산

배열의 전체 메모리 크기는 `요소 크기 * 배열 길이`로 계산된다.

```go
var a [100]int64   // 8바이트 * 100 = 800바이트
var b [100]int32   // 4바이트 * 100 = 400바이트
var c [100]bool    // 1바이트 * 100 = 100바이트
```

### 배열은 값 타입

Go에서 배열은 **값 타입**이다. 배열을 다른 변수에 대입하거나 함수에 전달하면 **전체가 복사**된다.

```go
a := [3]int{1, 2, 3}
b := a       // 배열 전체가 복사된다
b[0] = 99

fmt.Println(a) // [1 2 3] (원본 변경 안 됨)
fmt.Println(b) // [99 2 3]
```

이 동작은 의도치 않은 수정을 방지한다는 장점이 있지만, 큰 배열을 복사할 때 성능 문제가 발생할 수 있다.

```go
func processArray(arr [1000000]int) {
    // arr은 복사본이다. 100만 개의 int가 복사된다! (약 8MB)
}
```

> **참고:** 큰 배열을 함수에 전달할 때 복사 비용이 크므로, 실무에서는 **슬라이스(slice)**를 더 많이 사용한다. 슬라이스는 내부적으로 배열을 참조하므로 복사 비용이 없다. 또는 배열의 포인터를 전달하는 방법도 있다. 슬라이스는 다음 장에서 자세히 배운다.

## 10.4 다중 배열 (2차원 배열)

배열의 요소가 배열인 경우, 다중 배열(다차원 배열)이 된다. 행렬, 게임 보드, 이미지 데이터 등을 표현할 때 사용된다.

### 2차원 배열 선언과 초기화

```go
// 3행 4열 배열
var matrix [3][4]int

// 초기화와 함께
grid := [2][3]int{
    {1, 2, 3},
    {4, 5, 6},
}
```

`[2][3]int`는 "크기 3의 int 배열"을 요소로 가지는 "크기 2의 배열"이다. 즉, 바깥 배열의 크기가 행(row)의 수이고, 안쪽 배열의 크기가 열(column)의 수이다.

### 요소 접근

```go
grid := [2][3]int{
    {1, 2, 3},
    {4, 5, 6},
}

fmt.Println(grid[0][0])  // 1 (0행 0열)
fmt.Println(grid[1][2])  // 6 (1행 2열)

grid[0][1] = 99          // 0행 1열을 99로 변경
```

### 2차원 배열 순회

```go
matrix := [3][3]int{
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9},
}

// 인덱스 사용
for i := 0; i < len(matrix); i++ {
    for j := 0; j < len(matrix[i]); j++ {
        fmt.Printf("%d ", matrix[i][j])
    }
    fmt.Println()
}

// range 사용 (더 Go다운 방식)
for _, row := range matrix {
    for _, val := range row {
        fmt.Printf("%d ", val)
    }
    fmt.Println()
}
```

### 메모리 레이아웃

2차원 배열도 메모리에는 1차원으로 연속 배치된다. 행 우선(row-major) 순서로 저장된다.

```
[2][3]int:
         행0          행1
+----+----+----+----+----+----+
| 1  | 2  | 3  | 4  | 5  | 6  |
+----+----+----+----+----+----+
[0][0][0][1][0][2][1][0][1][1][1][2]
```

이 레이아웃을 이해하면 캐시 효율을 최적화할 수 있다. 행 단위로 순회하는 것이 열 단위로 순회하는 것보다 캐시 히트율이 높다.

```go
// 캐시 효율이 좋다 (행 우선 순회)
for i := 0; i < rows; i++ {
    for j := 0; j < cols; j++ {
        process(matrix[i][j])
    }
}

// 캐시 효율이 나쁘다 (열 우선 순회)
for j := 0; j < cols; j++ {
    for i := 0; i < rows; i++ {
        process(matrix[i][j])
    }
}
```

## 핵심 요약

| 항목 | 설명 |
|------|------|
| `var a [5]int` | 크기 5의 int 배열을 선언한다 |
| `[...]int{1,2,3}` | 크기를 자동 계산한다 |
| `len(arr)` | 배열 길이를 반환한다 |
| 인덱스 | 0부터 시작하며, 범위 초과 시 패닉이 발생한다 |
| 값 타입 | 대입/전달 시 전체가 복사된다 |
| 연속 메모리 | 빠른 접근과 높은 캐시 효율을 제공한다 |
| `[3][4]int` | 3행 4열 2차원 배열이다 |
| `range` | 배열 순회에 권장되는 방식이다 |

10. 4x4 2차원 배열을 선언하고, 행과 열을 전치(transpose)하는 프로그램을 작성하라. 원본 배열과 전치 결과를 모두 출력하라.

## 구현 과제

### 과제 1: 버블 정렬
10개의 정수 배열을 선언하고, 버블 정렬 알고리즘을 직접 구현하여 오름차순으로 정렬하라. 각 단계(pass)마다 배열의 상태를 출력하여 정렬 과정을 시각화하라.

### 과제 2: 행렬 곱셈
2x3 행렬과 3x2 행렬을 선언하고, 두 행렬의 곱을 계산하여 결과(2x2 행렬)를 출력하는 프로그램을 작성하라. 행렬 곱셈 공식: `C[i][j] = A[i][0]*B[0][j] + A[i][1]*B[1][j] + A[i][2]*B[2][j]`

### 과제 3: 선형 검색과 이진 검색
정렬된 20개의 정수 배열을 선언하라. 선형 검색(linear search)과 이진 검색(binary search)을 각각 구현하고, 특정 값을 찾는 데 각 방법이 몇 번의 비교를 수행하는지 출력하라. 두 알고리즘의 효율성 차이를 확인하라.

### 과제 4: 마방진 검증기
3x3 배열에 숫자를 채운 후, 그것이 마방진(magic square)인지 검증하는 프로그램을 작성하라. 마방진의 조건: 각 행의 합, 각 열의 합, 두 대각선의 합이 모두 같아야 한다. 1~9까지의 숫자를 한 번씩만 사용해야 한다.

### 과제 5: 이미지 회전 시뮬레이션
NxN 2차원 배열을 시계 방향으로 90도 회전하는 프로그램을 작성하라. 4x4 배열로 테스트하고, 원본과 회전 결과를 출력하라. (힌트: 회전 후 `result[j][N-1-i] = original[i][j]`)

## 프로젝트 과제

### 프로젝트 1: 틱택토(Tic-Tac-Toe) 게임
3x3 배열을 사용하여 틱택토 게임을 구현하라. 다음 기능을 포함하라:
- 게임 보드를 3x3 배열로 표현 (빈 칸, X, O)
- 매 턴마다 보드 상태를 보기 좋게 출력
- 두 플레이어가 번갈아 수를 놓는 시뮬레이션 (미리 정의한 수순 배열 사용)
- 이미 놓인 자리에 놓으려 하면 에러 메시지 출력
- 승리 조건 확인 (가로, 세로, 대각선)
- 무승부 판정 (보드가 꽉 찬 경우)
- 게임 종료 시 결과 메시지 출력

### 프로젝트 2: 학생 성적 관리 시스템
5명의 학생에 대해 5과목의 점수를 2차원 배열로 관리하는 프로그램을 작성하라. 다음 기능을 모두 구현하라:
- 학생별 총점과 평균 계산
- 과목별 총점과 평균 계산
- 학생별 석차(등수) 계산 (평균 기준 내림차순)
- 각 과목의 최고점/최저점과 해당 학생 번호 출력
- 전체 학생의 과목별 성적 분포 출력 (A/B/C/D/F 비율)
- 결과를 표 형식으로 보기 좋게 출력하라
- 학생 이름은 별도의 문자열 배열로 관리하라
